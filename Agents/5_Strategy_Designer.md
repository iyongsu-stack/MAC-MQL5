## Goal
**오프라인 마이닝(패턴 사전 구축)** + **온라인 매칭(실시간 벡터 쿼리)** 아키텍처를 설계하고, 피처 중요도 기반 랏사이즈 결정 모듈을 통합합니다.

> [!IMPORTANT]
> **원본 CSV 직접 로드 금지.** 모든 입력 데이터는 `1_Data_Prep`이 생성한 Parquet 파일을 사용합니다.
> 
> **🚨 AI 전략 개발 3대 핵심 원칙 (절대 준수)**
> 1. **Shift+1 원칙**: 실시간 온/오프라인 모델 로직 설계 시, M1 이외의 TF는 Shift+1 보정 로직 필수 삽입
> 2. **Friction Cost 30포인트**: EA가 실행하는 기대 수익 시나리오에 편도 마진이 아닌 마찰비용(스프레드+커미션) 환산 30포인트 사전 필터링 적용
> 3. **절대값 활용 금지**: EA의 판단 변수를 실시간 환율이나 금리 레벨 그 자체를 쓰지 말고, 동적 Z-Score로 모듈화할 것

## 실시간 트레이딩 아키텍처 (2026-02-24 도입)

> 상세 프레임워크: `Docs/TrendTrading Development Strategy/XAUUSD_AI_전략개발_종합_로드맵.md` 참조.

### 오프라인 단계 (서버 / 장 종료 후 또는 주말)
```
Walk-Forward 3단계 검증을 거쳐 뽑아낸
최정예 핵심 패턴(Centroid Vector) 10~20개를
"승률 사전" 형태로 벡터 DB(ChromaDB)에 저장

  패턴 A: [지표1, 지표2, ...] → 기대승률 82%, 핵심피처: BWMFI+CVD
  패턴 B: [지표1, 지표2, ...] → 기대승률 25% (위험 패턴)
```

### 온라인 단계 (실시간 M1 봉 — MT5 EA / Python)
```
1분봉 마감
    ↓
핵심 피처(3~5개) 값만 빠르게 수집 → 1개의 벡터 생성
    ↓
벡터 DB에 쿼리: "20개 패턴 중 가장 비슷한 것 1개 찾아와"
    ↓  (0.01초)
"현재 차트는 [패턴 A]와 92% 일치. 기대승률 82%"
    ↓
승률 > 60% → 즉시 진입 (랏사이즈는 점수에 비례)
승률 < 50% → Skip (대기)
```

## 피처 중요도 기반 랏사이즈 결정 (2026-02-24 추가)

> "AI는 방향을 결정하고, 피처 중요도 가중 점수는 그 결정에 얼마나 베팅할지를 결정한다."

### 종합 점수 → 랏사이즈 매핑

| 종합 점수 | 의미 | 랏사이즈 |
|---------|------|---------|
| 80% 이상 | 거의 모든 신호 일치 | **풀 사이즈** |
| 60~80% | 대부분 일치, 일부 불확실 | **하프 사이즈** |
| 40~60% | 신호 혼재 | **쿼터 사이즈** |
| 40% 미만 | 신호 불일치 | **거래 안 함** |

### 통합 시스템 전체 흐름
```
STEP 1: 현재 시간 확인 → 세션 필터 (런던/뉴욕만)
STEP 2: 현재 피처 벡터 생성 (핵심 피처 3~5개)
STEP 3: 벡터 DB에서 가장 일치하는 패턴 매칭 (0.01초)
STEP 4: 기대승률 > 60%? → 진입 결정
STEP 5: 피처 중요도 점수 계산 → 랏사이즈 결정
STEP 6: 진입 + Triple Barrier 설정
STEP 7: 자동 청산 (TP / SL / 시간 만료)
STEP 8: 거래 결과를 벡터 DB에 저장 → 에이전트 메모리 업데이트
```

### 안전 규칙 (절대 우선)
1. **상관관계 붕괴 시 → 랏사이즈 강제 0**: 금-달러 상관계수가 -0.2 이상이면 거래 중단
2. **연속 손절 시 → 자동 축소**: 3연패 이후 최소 랏(0.01)으로 강제 전환
3. **절대 한도**: 한 거래에서 계좌의 **2%를 초과하는 손실 위험** 노출 금지
4. **뉴스 이벤트 시 → 랏사이즈 0**: FOMC, NFP, CPI 발표 전후 30분 강제 거래 중단

### 접근법 A: 메타 라벨링 (1차 신호 → 2차 AI 필터) ⭐ 권장
```

           ↓ 신호 발생
AI 메타 라벨: "이 신호를 지금 따를 가치가 있는가?"
           ↓ Yes
피처 중요도 → 랏사이즈 결정 후 진입
```

### 접근법 B: 확률 분포 예측 (중급)
```
AI 출력: "TP 먼저 도달 확률 = 63%"
         → 60% 이상이면 진입
         → 70% 이상이면 풀사이즈 진입
```
