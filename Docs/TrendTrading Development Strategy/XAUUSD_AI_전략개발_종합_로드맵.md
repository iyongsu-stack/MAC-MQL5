> **🚨 AI 전략 개발 3대 핵심 원칙 (모든 단계에 예외 없이 강제 적용)**
> 1. **Shift+1 원칙**: 시계열 병합, 시그널 로직 등 모든 단계에서 상위 프레임 참조 시 직전 완성봉만 확인 (Look-ahead Bias 0% 허용).
> 2. **Friction Cost 30포인트**: 오프라인 분석/모의 백테스트 등 모든 수익 기대값 판정 시 보수적 마찰 비용 30포인트를 디폴트로 차감 처리.
> 3. **절대값 사용 금지**: 거래 트리거나 컷오프를 원본 절대수치가 아닌 동적 정규화 값(Z-score 등)에 의존하도록 개발.

# XAUUSD AI 전략 개발 종합 로드맵

현재 진행 중인 전략 개발 방식과 지금까지 논의된 내용을 종합하여, 앞으로의 개발 방향을 정리한 문서입니다.

> [!IMPORTANT]
> **2026-02-24 주요 업데이트:** 기존 "인간 주도(Bottom-Up) 규칙 도출" 방식에서 **"AI 주도(Top-Down) 패턴 마이닝"** 방식으로 전환. 3단계 Walk-Forward 검증, 메가 피처 풀 투입, 오프라인/온라인 분리 아키텍처 도입. 상세 프레임워크: [[XAUUSD_AI_하이브리드_퀀트_프레임워크_종합]] 참조.

## 핵심 사상

> **"인간은 직관(라벨)만 제공하고, AI가 핵심 피처와 패턴을 스스로 발라낸다. 실전에서는 검증된 패턴과의 일치도만 보고 0.01초 만에 진입한다."**

> [!IMPORTANT]
> ### 🎯 확정된 전략 구조 (2026-02-27)
> ```
> ┌─────────────────────────────────────────────────────────┐
> │  사용자(규칙)가 정하는 것 — 딱 1개                        │
> │                                                         │
> │  Setup: LRAVGST_Avg(180)_BSPScale > 1.0                │
> │         (황금 구간 — 이 조건에서만 진입 후보로 고려)       │
> └─────────────────────────────────────────────────────────┘
>                           ↓
> ┌─────────────────────────────────────────────────────────┐
> │  AI가 480개 피처로 학습하는 것                             │
> │                                                         │
> │  ✔ 눌림목인지 아닌지                                     │
> │  ✔ 너무 늦은 타이밍인지 아닌지                            │
> │  ✔ 지금 쏠지 말지 (진입 여부 최종 결정)                   │
> └─────────────────────────────────────────────────────────┘
>                           ↓
> ┌─────────────────────────────────────────────────────────┐
> │  TrailingStopVx 전담                                    │
> │  → 청산 (고정 TP 사용 안 함, 추세를 끝까지 추적)           │
> └─────────────────────────────────────────────────────────┘
> ```


```
기존 방식 (인간 주도 - Bottom-Up):
  인간이 규칙 도출 (if RSI < 30 and ADX > 20...)
  → 인간이 피처 1개씩 추가/테스트
  → 느리고, 주관적이며, 편향에 취약

새로운 방식 (AI 주도 - Top-Down):                         ⬅️ 채택
  인간은 "여기가 좋아 보여" 라벨만 찍음
  → 모든 피처(기술+매크로)를 처음부터 전부 투입
  → AI(LightGBM + SHAP)가 알아서 핵심 피처를 추출하고 패턴화
  → 빠르고, 객관적이며, 확장 가능
```

## 전체 로드맵 요약

```
[현재 위치]

Phase 0 ← 지금 여기
  수동 라벨링 + 규칙 도출
  "내 눈으로 본 좋은 진입점의 규칙을 찾는 중"

      ↓ 다음 단계들

[앞으로의 로드맵]

Step 1: 패턴 마이닝 (최근 2개월 데이터)
  수동 라벨링 → Triple Barrier 객관화 → 자동 확장 → AI 패턴 추출
  메가 피처 풀(기술+매크로+세션) 전부 투입
  SHAP으로 핵심 피처 3~5개 자동 추출 + 핵심 패턴(Centroid) 도출

Step 2: 모의고사 검증 (1년 데이터)
  Step 1의 패턴을 과거 1년 치에 적용 → Fail-Fast

Step 3: 최종 실전 검증 (10년 데이터)
  10년 백테스트 통과 시 → 실전 봇(EA) 투입

Phase 5: 소액 실전 + 에이전트 메모리 시스템
```

> [!CAUTION]
> **절대 원칙: Triple Barrier 라벨링(정답지 확정)은 반드시 AI 패턴 마이닝보다 선행되어야 합니다.**
> 인간의 주관적 라벨(후행 편향 포함)을 그대로 AI에 밀어 넣으면, AI는 편향된 패턴을 외워버려 실전에서 계좌를 날립니다(GIGO). Triple Barrier로 냉혹하게 채점한 "검증된 정답지"만이 AI 학습의 유일한 원료입니다.

## Step 1: 패턴 마이닝 (Training — 최근 2개월)

> **목표:** 인간의 직관적 라벨 + Triple Barrier 객관화 + AI 패턴 마이닝으로 핵심 피처와 패턴을 자동 추출

### 1-1. 인간 가이드 — 수동 라벨링

차트를 보며 "여기가 좋은 진입점 같다" 직관으로 100~200개를 라벨링합니다.

```
라벨링할 때 반드시 함께 기록:
  ✅ 진입 근거: M5 iBWMFI 양수 + CVD 상승 전환 등 (구체적 수치)
  ✅ H4 ADX 값: 25 (추세 진행 중)
  ✅ 진입 후 최대 역행: 진입 후 -$0.50까지 역행한 뒤 반등
  ✅ 진입 후 TP 도달까지 시간: 18분 소요
  ✅ 그 시점의 세션: 런던 세션 16:42
  ✅ ATR(14): $0.85
```

> [!WARNING]
> **후행적 편향(Look-ahead Bias) 위험:** 과거 차트를 사후에 보며 라벨링하면, "그때 알 수 없었던 미래 정보"가 무의식적으로 반영됩니다. 이 편향은 다음 단계(Triple Barrier)에서 냉혹하게 제거됩니다.

### 1-2. 냉혹한 채점 — ATR 동적 배리어 객관화 (학습 전용)

컴퓨터가 각 라벨 시점의 60분 뒤 미래를 보고 롱/숏 각각 "이건 수익(1), 이건 손실(0)" 객관적 정답지를 확정합니다.
**하이브리드 아키텍처**: AI는 "진입만" 학습. 실전 청산은 TrailingStopVx가 전담.

```
ATR 동적 배리어 설정 (학습용 진입 타이밍 정답지):
  TP:     ATR(14) × 1.5  (변동성 대비 도 유의미한 방향성 관성 판별)
  SL:     ATR(14) × 1.0  (노이즈 필터링, TP보다 작게 → 비대칭 R:R)
  시간:   60분 (60봉)   (세션 내 방향 확립 주기)
  Friction: 30pt 차감 (IC Markets 스프레드+슬리피지, A2 원칙)
  라벨: label_long(1/0), label_short(1/0) 분리
```

> [!CAUTION]
> **거래 비용 미반영 리스크:** 스프레드($0.10~$0.30) + 슬리피지($0.05~$0.15)를 라벨링 시 반드시 차감해야 합니다. 비용을 빼지 않으면 "수익"으로 분류된 타점이 실전에서 "손실"로 뒤집힙니다.

> [!WARNING]
> **M1 섀도우 겹침(Shadow Overlap) 문제:** 변동성 큰 구간에서 단 1개의 M1 봉 안에서 TP와 SL 범위를 모두 터치할 수 있습니다. Triple Barrier 라벨링은 반드시 **Tick 데이터(Every tick based on real ticks)** 기반으로 수행해야 합니다.

### 1-3. 자동 라벨 확장

> [!IMPORTANT]
> **라벨 부족 → 과적합 리스크:** 수동 라벨 200개만으로 30+개의 피처를 학습시키면 AI가 우연한 패턴을 진짜로 착각합니다(차원의 저주). 반드시 자동 확장이 필요합니다.

```
수동 라벨 200개 → 전체 1분봉(약 43,200개)에 자동 ATR 동적 배리어 적용
→ 학습 데이터가 수만 건으로 폭증
→ AI가 고차원 피처에서 의미 있는 패턴을 찾을 수 있는 충분한 데이터 확보
```

### 1-4. 메가 피처 풀 투입 — 모든 데이터를 처음부터 전부 넣는다

> **"AI가 100개 중 97개를 쓰레기로 버리고, 진짜 범인 3개만 찾아낸다."**

학습 테이블 구조 (1개 Row = 특정 시점의 벡터):

| 구분 | 피처 예시 | 변환 방법 |
|:---|:---|:---|
| **정답(Y)** | ATR 동적 배리어 결과 — **롱: label_long(1/0), 숏: label_short(1/0) 분리** | Friction Cost 30pt 차감 후 방향별 판정 |
| **기술 피처(X1~X20)** | M5 BWMFI, M5 CVD, M1 ATR, H4 ADX, RSI, Chaikin, QQE 등 | 멀티스케일 Z-score (60/240/1440) |
| **매크로 피처(X21~X40)** | UST10Y, EURUSD, US500, DXY, VIX, XTIUSD, 금리차 등 | **변화율(Δ%)** 또는 H1 윈도우 평균 |
| **세션 피처(X41)** | Asia / London / NewYork | One-hot 또는 sin/cos 인코딩 |
| **시간 피처(X42~X43)** | 요일, 시간대 | 주기적 인코딩 (sin/cos) |

> [!WARNING]
> **타임프레임 불일치 리스크:** M1 지표는 1분마다 바뀌지만 금리(UST10Y)는 하루에 한 번 바뀝니다. 매크로 피처를 **변화율(Δ%)** 또는 H1/H4 윈도우 평균으로 사전 변환하지 않으면 AI가 금리를 "정적(=중요하지 않음)"으로 오인합니다.

IC Markets에서 매크로 피처로 사용할 수 있는 전체 심볼 목록은 [[XAUUSD_AI_피처_완전_가이드]]의 IC Markets 심볼 활용 지도 및 [[IcMarketTradingIndex]] 참조.

### 1-5. AI 패턴 마이닝 — 핵심 피처 자동 추출

```
정답지(Y) + 모든 피처(X: 기술+매크로+세션) 통째로
머신러닝 모델(LightGBM)에 투입

      ↓

SHAP 분석으로 핵심 피처 3~5개 자동 추출
"승률 75%를 만든 1등 공신은 [H4 ADX]였고, 기여도 35%다"
"2등은 [M5 BWMFI] 20%다"

      ↓

상위 핵심 피처만 유지, 하위 80% 피처 제거
핵심 패턴(Centroid Vector) 도출 → 벡터 DB에 등록
```

### 핵심 피처를 골라내는 3가지 기법

| 기법 | 방법 | 용도 | 적용 시점 |
|:---|:---|:---|:---|
| **방법 A: 차원별 거리 분해** | 벡터 DB 검색 후 Numpy로 피처별 편차 계산 | 실시간 모니터링 콘솔 | 실시간 |
| **방법 B: SHAP XAI 분석** ⬅️ 주력 | LightGBM + SHAP으로 피처 기여도 역추적 | 오프라인 리포트 — "핵심 피처 순위" | 오프라인 |
| **방법 C: 피처 마스킹** | 특정 지표를 0으로 마스킹 후 결과 변화 관찰 | 새 지표 유효성 검증 | 오프라인 연구용 |

> [!NOTE]
> **다중공선성(Multicollinearity) 주의:** 상관관계가 높은 지표끼리(예: UST10Y 변화율과 EURUSD 변화율) 서로 기여도를 분산시켜 SHAP 결과를 왜곡합니다. **상관계수 > 0.85인 피처 쌍 중 하나를 제거**하는 사전 필터링을 적용하세요.

## Step 2: 모의고사 검증 (Validation — 1년)

> **목표:** Step 1에서 찾은 패턴이 "2달치의 우연"이 아님을 입증

```
Step 1에서 찾은 핵심 패턴(알고리즘)을 과거 1년 치 데이터에 적용

  승률 유지됨?
    YES → Step 3으로 이동 (2달치의 우연이 아님을 입증)
    NO  → 모델 즉각 폐기, Step 1으로 돌아가 재시도 (Fail-Fast)
```

> [!IMPORTANT]
> **10년 데이터 스케일 왜곡 리스크 대비:** 2016년 금값 $1,200 vs 2024년 $2,400 — 같은 ATR 1.0이라도 절대가격에 따라 의미가 다릅니다. 1년 검증 단계부터 반드시 모든 피처를 **멀티스케일 Z-score(60/240/1440) 또는 비율(%)**로 정규화하세요.

## Step 3: 최종 실전 검증 (Out-of-Sample — 10년)

> **목표:** 모든 거시경제 이벤트(코로나, 금리 인상, 양적 완화 등)를 견뎌내는 로버스트한 패턴인지 최종 확인

```
10년 치 데이터 (다양한 거시경제 이벤트 포함 구간)

  PnL 곡선 우상향 + MDD 견딜 수 있는 수준?
    YES → 최종 전략 채택 → 실전 봇(EA) 투입 + 소액(0.01랏) 실전 시작
    NO  → 모델 폐기
```

> [!WARNING]
> **생존자 편향(Survivorship Bias):** 10년 백테스트 수익이 특정 1~2년의 대박에 의해 평균이 왜곡될 수 있습니다. 반드시 **연도별 수익을 분해**하여 모든 연도에서 꾸준히 양수인지 확인하세요.

## 실시간 트레이딩 아키텍처: 오프라인 마이닝 + 온라인 매칭

개발(비동기)과 실전(실시간)을 완벽히 분리하는 초지연(Ultra-Low Latency) 시스템.

### 오프라인 단계 (서버 / 메인 컴퓨터 — 장 종료 후 또는 주말)

```
3단계 Walk-Forward 검증을 거쳐 뽑아낸
최정예 핵심 패턴(Centroid Vector) 10~20개를
"승률 사전" 형태로 벡터 DB(ChromaDB)에 저장

  패턴 A: [지표1값, 지표2값, ...] → 기대승률 82%, 핵심피처: BWMFI+CVD
  패턴 B: [지표1값, 지표2값, ...] → 기대승률 25% (위험 패턴)
  패턴 C: ...
```

### 온라인 단계 (실시간 M1 봉 — MT5 EA / Python)

```
1분봉 마감
    ↓
핵심 피처(3~5개) 값만 빠르게 수집 → 1개의 벡터 생성
    ↓
벡터 DB에 쿼리: "20개 패턴 중 가장 비슷한 것 1개 찾아와"
    ↓  (0.01초)
"현재 차트는 [패턴 A]와 92% 일치. 기대승률 82%"
    ↓
승률 > 60% → 즉시 진입 (랏사이즈는 승률에 비례)
승률 < 50% → Skip (대기)
```

### 패턴 사전 관리 및 갱신 정책

```
패턴 사전 유효기간: 최대 3개월

갱신 트리거:
  ① 정기: 3개월마다 전체 재마이닝 (Walk-Forward 3단계 재실행)
  ② 긴급: 최근 20거래 승률 < 40% → 즉시 경고 → 재마이닝
  ③ 이벤트: 구조적 변화 감지 시 (금리 방향 전환 등)

갱신 절차:
  최근 2개월 데이터로 Step 1 재실행
      ↓
  1년 검증 (Step 2)
      ↓
  통과 시 새 패턴 사전으로 교체
```

> [!CAUTION]
> **패턴 사전 노후화 리스크:** 시장 구조가 변하면(예: 금리 인하→인상 전환) 기존 패턴이 완전히 무효화됩니다. 3개월 주기 정기 재마이닝 + 실시간 승률 모니터링(40% 이하 자동 경고)이 필수입니다.

## Phase 5: 피처 중요도 랏사이즈 + 소액 실전 + 에이전트 메모리 시스템

> **목표:** 전체 시스템을 통합하고 소액으로 실전 검증, 에이전트 메모리 시스템으로 적응적 학습

### 통합 시스템 전체 흐름

```
STEP 1: 현재 시간 확인 → 세션 필터 (런던/뉴욕만)
STEP 2: 현재 피처 벡터 생성 (핵심 피처 3~5개)
STEP 3: 벡터 DB에서 가장 일치하는 패턴 매칭 (0.01초)
STEP 4: 기대승률 > 60%? → 진입 결정
STEP 5: 피처 중요도 점수 계산 → 랏사이즈 결정
STEP 6: 진입 + Triple Barrier 설정
STEP 7: 자동 청산 (TP / SL / 시간 만료)
STEP 8: 거래 결과를 벡터 DB에 저장 → 에이전트 메모리 업데이트
```

### 에이전트 메모리 시스템 (FINMEM 스타일)

```
메모리 계층 구조:

  Layer 1 (단기 메모리): 최근 100거래 결과
    → 현재 전략의 실시간 성과 모니터링
    → "최근 10거래 중 7거래 손실" → 자동 경고

  Layer 2 (중기 메모리): 최근 1개월 국면별 성과
    → "금리 하락 국면에서 승률 72%, 상승 국면에서 45%"
    → 국면별 전략 가중치 자동 조정

  Layer 3 (장기 메모리): 전체 거래 이력의 패턴 DB
    → 과거 최적화 파라미터 이력
    → 성공/실패 거래의 공통 패턴
    → 구조적 변화(Structural Break) 이력
```

### 소액 실전 규칙

```
단계 1: 0.01 랏으로 1개월 (최소 100거래 이상)
단계 2: 0.03 랏으로 1개월
단계 3: 0.05 랏으로 1개월
     → 각 단계에서 Profit Factor > 1.2 유지 시만 다음 단계 진행

절대 규칙:
  • 한 거래 최대 손실 = 계좌의 2%
  • 일일 최대 손실 = 계좌의 5%
  • 연속 3회 손절 → 당일 거래 중단
  • 주간 최대 손실 = 계좌의 10% → 1주 일시 중단
```

## 전체를 관통하는 핵심 원칙

> [!IMPORTANT]
> **1. "얼마나 잘 맞추느냐"보다 "얼마나 안 잃느냐"가 중요**
> 진입 승률 55%라도 리스크 관리가 완벽하면 장기적으로 수익.
> 승률 70%라도 한 번의 대형 손실이 전부 날림.
>
> **2. Triple Barrier 라벨링(객관 정답지)은 절대로 건너뛰지 말 것**
> 인간의 주관적 라벨을 그대로 AI에 넣으면 GIGO(Garbage In, Garbage Out).
>
> **3. Walk-Forward 검증 없이 실전 투입하지 말 것**
> 2달→1년→10년 3단계 검증을 모두 통과한 패턴만 실전에 투입.
>
> **4. 베이스라인 먼저 측정할 것**
> 가장 단순한 규칙(예: `M5 BWMFI > 0`)의 승률을 먼저 구하고, AI 모델이 이보다 확실히 우수해야 합니다.

## 전체 타임라인 요약

```
Phase 0 (현재 진행 중): 수동 라벨링 + 규칙 도출
  → 즉시: 객관적 기록 추가 + 성과 역산 시작
  → 벡터 DB 사전 조사 (DB 선택, 참고 자료 실습)

Step 1 (4~6주): 패턴 마이닝 (최근 2개월)
  → 수동 라벨 → Triple Barrier 객관화 → 자동 확장
  → 메가 피처 풀(기술+매크로) 전부 투입
  → AI(LightGBM + SHAP) 핵심 피처 추출 + 패턴 도출
  → 벡터 DB(ChromaDB) 패턴 사전 등록

Step 2 (2~3주): 모의고사 검증 (1년)
  → Step 1 패턴을 1년 치 데이터에 적용
  → 승률 유지 → Step 3 / 붕괴 → Step 1 재시도

Step 3 (2~3주): 최종 실전 검증 (10년)
  → 10년 치 백테스트 + 연도별 수익 분해 확인
  → 통과 시 실전 투입

Phase 5 (3개월 이상): 소액 실전 + 자금 확대
  → 0.01 랏부터 단계적 증액
  → 에이전트 메모리 시스템 (FINMEM 스타일 계층적 메모리)
  → 3개월마다 패턴 사전 재마이닝

전체 예상 기간: 약 4~6개월
```

## 리스크·위험 요인 및 대응 방안 종합

| # | 리스크 | 설명 | 대응 방안 |
|:---:|:---|:---|:---|
| 1 | **라벨 부족 → 과적합** | 수동 라벨 200개로 30+개 피처 학습 시 우연한 패턴을 진짜로 착각 | Triple Barrier로 전체 1분봉(수만 건)에 자동 라벨 확장 |
| 2 | **타임프레임 불일치** | M1 지표와 금리(일 단위)를 같은 Row에 넣으면 AI가 금리를 정적으로 오인 | 매크로 피처를 **변화율(Δ%)**이나 H1/H4 윈도우 평균으로 사전 변환 |
| 3 | **10년 데이터 스케일 왜곡** | 2016년 금값 $1,200 vs 2024년 $2,400 — 같은 ATR이라도 의미가 다름 | 모든 피처를 **멀티스케일 Z-score(60/240/1440) 또는 비율(%)**로 정규화 |
| 4 | **패턴 사전 노후화** | 3개월 전 패턴이 시장 구조 변화로 무효화 | 3개월 주기 정기 재마이닝 + 실시간 승률 모니터링(40% 이하 경고) |
| 5 | **거래 비용 미반영** | 스프레드·슬리피지를 라벨에 미반영 시 실전 손실 전환 | Triple Barrier 시 **IC Markets 스프레드 + 슬리피지** 차감 |
| 6 | **후행적 편향** | 과거 차트를 사후에 보며 라벨링하면 미래 정보 무의식 반영 | Triple Barrier 객관 채점 + Walk-Forward 3단계 기간 완전 분리 |
| 7 | **과도한 피처 수** | 100+개 피처를 넣으면 모델 수렴 불안정 + 잡음 패턴 학습 | SHAP/Permutation Importance로 **상위 10~15개만 유지** |
| 8 | **다중공선성** | 상관관계 높은 지표 쌍이 피처 중요도를 왜곡 | 상관계수 > 0.85인 피처 쌍 중 하나를 **사전 필터링** 제거 |
| 9 | **데이터 품질 오류** | 결측치·이상치가 벡터에 포함되면 거리 계산 왜곡 | ETL 파이프라인에 **정합성 검증 단계** 삽입 |
| 10 | **벡터 DB 조회 지연** | 실시간 매칭 시 0.1초 이상 걸리면 트레이딩 기회 상실 | 패턴 사전을 **10~20개 정예**로 유지 + HNSW 인덱스 최적화 |
| 11 | **생존자 편향** | 10년 백테스트 수익이 특정 1~2년 대박에 의해 왜곡 | **연도별 수익 분해** 확인 — 모든 연도에서 꾸준히 양수인지 검증 |
