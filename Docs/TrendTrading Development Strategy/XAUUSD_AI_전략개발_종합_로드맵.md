# XAUUSD AI 전략 개발 종합 로드맵

현재 진행 중인 전략 개발 방식과 지금까지 논의된 내용을 종합하여, 앞으로의 개발 방향을 정리한 문서입니다.

> [!IMPORTANT]
> **2026-03-11 주요 업데이트:** AI 주도(Top-Down) 패턴 마이닝 방식 채택. 4계층 데이터 파이프라인 구축 완료 (`AI_Study_Dataset.parquet` — 817컬럼, 266만행, 2018-08~2026). **AI 학습 A+B+C 통합 모델 완료** (AUC=0.8298, OOS 3/3 PASS). 실전 조합: M30_DiPlus>35 + thr=0.25 → **승률 56.3%, 월 ~15건**. **피라미딩 V2 라벨링 및 Model_AddOn 학습 진행 중** (Phase 3.5). 워크플로우: `/data-build`, `/ai-train`, `/labeling` 참조.

## 🎯 프로젝트 전략 목표

1. **거래 횟수**: 롱+숏+Reversal 통틀어 **주 3~4회**
2. **전략 구성**: 3개 전략 개별 개발 후 통합
3. **목표**: 높은 승률(70~80%) + 큰 보장 수익(월 20%) + 리스크 1% 이하
4. **철학**: 진입 빈도 제한을 감수하더라도 확실한 승률 구간에만 진입하며, 수익 극대화는 **피라미딩(추세 진행 중 추가 진입)**으로 해결한다.
5. **피라미딩**: 1차 진입(Model_Entry)이 수익 중일 때, AI가 추가 진입 시점을 판단(Model_AddOn)하여 수익을 극대화 — **현재 Phase 3.5에서 라벨링 및 모델 학습 진행 중**

> 인간의 주관적 라벨(후행 편향 포함)을 그대로 AI에 밀어 넣으면, AI는 편향된 패턴을 외워버려 실전에서 계좌를 날립니다(GIGO). Triple Barrier로 냉혹하게 채점한 "검증된 정답지"만이 AI 학습의 유일한 원료입니다.

## Step 1: 패턴 마이닝 (Training — 최근 2개월)

> **목표:** 인간의 직관적 라벨 + Triple Barrier 객관화 + AI 패턴 마이닝으로 핵심 피처와 패턴을 자동 추출

### 1-1. 인간 가이드 — 수동 라벨링

차트를 보며 "여기가 좋은 진입점 같다" 직관으로 100~200개를 라벨링합니다.

```
라벨링할 때 반드시 함께 기록:
  ✅ 진입 근거: M5 iBWMFI 양수 + CVD 상승 전환 등 (구체적 수치)
  ✅ H4 ADX 값: 25 (추세 진행 중)
  ✅ 진입 후 최대 역행: 진입 후 -$0.50까지 역행한 뒤 반등
  ✅ 진입 후 TP 도달까지 시간: 18분 소요
  ✅ 그 시점의 세션: 런던 세션 16:42
  ✅ ATR(14): $0.85
```

> [!WARNING]
> **후행적 편향(Look-ahead Bias) 위험:** 과거 차트를 사후에 보며 라벨링하면, "그때 알 수 없었던 미래 정보"가 무의식적으로 반영됩니다. 이 편향은 다음 단계(Triple Barrier)에서 냉혹하게 제거됩니다.

### 1-2. 냉혹한 채점 — CE Trailing Stop 라벨링 (학습 전용)

컴퓨터가 각 라벨 시점의 미래를 보고 롱 "이건 수익(1), 이건 손실(0)" 객관적 정답지를 확정합니다.
**하이브리드 아키텍처**: AI는 "진입만" 학습. 실전 청산은 TrailingStopVx가 전담.

```
✅ 확정된 CE Trailing Stop 라벨링 설정 (2026-03-09 기준):
  TP:     ATR(14) × 2.0 이상 (CE Trailing Stop 기반)
  SL:     ATR(14) × 2.5  
  시간:   30분 (30봉)    (세션 내 방향 확립 주기)
  판정:   Close 기준 TP 도달 판정
  Friction: 30pt 차감 ($0.30, 스프레드+슬리피지)
  라벨:   label_long(1/0) 전용 (숏은 롱 완성 후 별도 학습)
  하드 필터: ADXMTF_M30_DiPlus > 35 (라벨링 외부, AI 모델 앞단 적용)
  IS Win Rate: 8.7% (전봉 기준) / OOS Win Rate: 10.6%
```

> [!CAUTION]
> **거래 비용 미반영 리스크:** Friction Cost 30pt를 라벨링 시 반드시 차감. 비용을 빼지 않으면 "수익"으로 분류된 타점이 실전에서 "손실"로 뒤집힙니다.

### 1-3. 자동 라벨 확장

> [!IMPORTANT]
> **라벨 부족 → 과적합 리스크:** 수동 라벨 200개만으로 30+개의 피처를 학습시키면 AI가 우연한 패턴을 진짜로 착각합니다(차원의 저주). 반드시 자동 확장이 필요합니다.

```
수동 라벨 200개 → 전체 1분봉(약 43,200개)에 자동 ATR 동적 배리어 적용
→ 학습 데이터가 수만 건으로 폭증
→ AI가 고차원 피처에서 의미 있는 패턴을 찾을 수 있는 충분한 데이터 확보
```

### 1-4. 메가 피처 풀 투입 — ✅ 구축 완료 (817컬럼)

> **"AI가 817개 중 757개를 쓰레기로 버리고, 진짜 범인 60개만 찾아낸다. 여기에 pct16+regime4를 더해 80개 최종."**

`AI_Study_Dataset.parquet` (266만행, 2018-08~2026) 확정 구조:

| 구분 | 피처 수 | 내용 | 파생 방법 |
|:---|:---:|:---|:---|
| **정답(Y)** | 2 | `label_long(1/0)`, `label_barrier_hit` | CE Trailing Stop, Friction $0.30 차감 |
| **기술 피처** | ~165 | BSP, ADX, QQE, TDI, CHOP, CE_Dist, ATR 등 | Z-score(60/240), Slope, Accel, 비율, **pct240/1440** |
| **MTF 파생** | ~50 | M5/H4 변화점 기반 Slope/Z-score | ffill 매핑 |
| **매크로 피처** | ~360 | DXY, VIX, UST10Y, BTC, FRED 금리·인플레 등 | Δ%, Z-score(60/240/1440), Slope |
| **세션/시간** | ~6 | 세션 원핫, 요일/시간 sin/cos | 계산 |
| **TickVolume** | ~6 | MA비율(60/240/1440), Z-score(60/240), **pct240** | 원본 DROP |

> [!NOTE]
> **핵심 파생 유형 (절대값 금지)**: 가속도, 멀티스케일 Z-Score(60/240/1440), 비율, Slope, 변화율(Δ%), 정규화 이격도, **롤링 퍼센타일 랭크(pct240/pct1440)**. 스케일 오실레이터(0~100)는 Passthrough 허용.
> **Z-score 한계 인식 ★**: 레짐 전환 시 스케일 불일치 발생 → pct 피처 병행 필수 (실측: OOS 승률 33%→52% 개선).

### 1-5. AI 패턴 마이닝 — ✅ 핵심 피처 자동 추출 완료

```
817개 피처(X: 기술+매크로+세션) 통째로 투입
      ↓
Spearman 상관 0.85 Pruning → 811→419개
      ↓
LightGBM Expanding Walk-Forward 5-Fold (평균 AUC 0.7967)
      ↓
SHAP Top-60 자동 선별 (pct 피처 16개 자연 선발)
      ↓
2라운드: Top-60 + 동적pct(+16) + 레짐피처(+4) = 80개
  + 단조 제약 (+1=19, -1=8) + FP 3배 페널티
  → AUC 0.8298, OOS 3/3 PASS
```

**SHAP Top-5 핵심 피처 (실측):**

| 순위 | 피처명 | SHAP | 해석 |
|:---:|:---|:---:|:---|
| 1 | `CE_SL2_dist_ATR` | 0.414 | CE2 SL까지의 거리 (핵심 신호) |
| 2 | `ADXMTF_H4_DiPlus_pct240` | 0.233 | H4 DiPlus 퍼센타일 (레짐 적응 핵심) |
| 3 | `CE_Dist2_slope14` | 0.376 | CE2 거리 기울기 |
| 4 | `QQE_(5-14)_TrLevel` | 0.313 | QQE 트렌드 레벨 |
| 5 | `ADXMTF_M5_DiPlus` | 0.205 | ADX M5 매수방향 강도 |

> [!NOTE]
> **다중공선성(Multicollinearity) 처리:** Spearman 상관계수 > 0.85인 피처 쌍을 사전 제거 (811→419개). 실측: Target MI 낮은 쪽 제거.
> **pct 피처의 핵심 역할:** `ADXMTF_H4_DiPlus_pct240`이 SHAP 전체 2위 — "H4 DiPlus가 최근 240봉 중 상위 몇%인가"가 레짐 적응의 핵심 신호.

## Walk-Forward 최종 검증 (OOS: 2023-01 ~ 2026)

> **목표:** 학습에 전혀 사용하지 않은 데이터(2023~2026)로 모델의 실전 생존력 검증

### ✅ A+B+C 모델 OOS 결과 (thr=0.20 기준, 승률≥45% PASS)

| 단계 | 검증 구간 | 승률 | 신호 수 | 판정 |

## 실시간 트레이딩 아키텍처: 오프라인 마이닝 + 온라인 매칭

개발(비동기)과 실전(실시간)을 완벽히 분리하는 초지연(Ultra-Low Latency) 시스템.

### 오프라인 단계 (서버 / 메인 컴퓨터 — 장 종료 후 또는 주말)

```
3단계 Walk-Forward 검증을 거쳐 뽑아낸
최정예 핵심 패턴(Centroid Vector) 10~20개를
"승률 사전" 형태로 벡터 DB(ChromaDB)에 저장

  패턴 A: [지표1값, 지표2값, ...] → 기대승률 82%, 핵심피처: BWMFI+CVD
  패턴 B: [지표1값, 지표2값, ...] → 기대승률 25% (위험 패턴)
  패턴 C: ...
```

### 온라인 단계 (실시간 M1 봉 — MT5 EA / Python)

```
1분봉 마감
    ↓
핵심 피처(3~5개) 값만 빠르게 수집 → 1개의 벡터 생성
    ↓
벡터 DB에 쿼리: "20개 패턴 중 가장 비슷한 것 1개 찾아와"
    ↓  (0.01초)
"현재 차트는 [패턴 A]와 92% 일치. 기대승률 82%"
    ↓
승률 > 60% → 즉시 진입 (랏사이즈는 승률에 비례)
승률 < 50% → Skip (대기)
```

### 패턴 사전 관리 및 갱신 정책

```
패턴 사전 유효기간: 최대 3개월

갱신 트리거:
  ① 정기: 3개월마다 전체 재마이닝 (Walk-Forward 3단계 재실행)
  ② 긴급: 최근 20거래 승률 < 40% → 즉시 경고 → 재마이닝
  ③ 이벤트: 구조적 변화 감지 시 (금리 방향 전환 등)

갱신 절차:
  최근 2개월 데이터로 Step 1 재실행
      ↓
  1년 검증 (Step 2)
      ↓
  통과 시 새 패턴 사전으로 교체
```

> [!CAUTION]
> **패턴 사전 노후화 리스크:** 시장 구조가 변하면(예: 금리 인하→인상 전환) 기존 패턴이 완전히 무효화됩니다. 3개월 주기 정기 재마이닝 + 실시간 승률 모니터링(40% 이하 자동 경고)이 필수입니다.

## 🔺 피라미딩 전략 — Phase 3.5 (현재 진행 중)

> **목표:** 1차 진입(Model_Entry) 이후 추세가 진행 중일 때, AI가 추가 진입 시점을 판단(Model_AddOn)하여 **수익을 30~50% 극대화**

### 피라미딩 핵심 원리

```
가격 ↑
  │         ┌──── 3차 추가 (0.25 랏)
  │     ┌───┤
  │ ┌───┤   └──── TrailingStop ━━━━━━━━━━┐
  │ │   └──── 2차 추가 (0.50 랏)            │ 
  │ │                                     │ 그룹 전체 일괄 청산
  ├─┤──────── 1차 진입 (1.00 랏)          │
  │ │                                     ▼
──┴─┴───────────────────────────────────────── 시간 →
```

**정피라미드(Standard)** 채택: 추세 초기에 가장 크게, 추가 진입마다 축소. 평균 단가 유리하고 리스크 최소화.

### V2 라벨링 설계 (label_pyramid)

기존 CE Trailing Stop 라벨(label_long)과 **별도의 피라미딩 전용 라벨**을 생성합니다:

```
라벨링 조건:
  대상: label_long=1(Win)인 원본 거래 궤적 내의 모든 봉
  제외: 눌림목 State Machine (running_max 대비 1.0×ATR 이상 하락 시 진입 차단)
  
  label_pyramid = 1 (추가 진입 성공):
    → CE2 발동으로 청산 시 수익 ≥ 2.0×ATR (Friction $0.30 차감 후)
  label_pyramid = 0 (추가 진입 실패):
    → SL 터치(-2.5×ATR) 또는 CE2 청산 수익 < 2.0×ATR
```

> [!IMPORTANT]
> **V2 핵심 개선**: 가격 구역 조건(zone_lower/zone_upper) 제거 → 눌림목 State Machine만으로 필터링. CE2 발동 시점 O(n) 사전 계산으로 고속 처리.

### 2-모델 구조 (Model_Entry + Model_AddOn)

```
┌──────────────────────┐
  M1 봉 데이터 ──→ │  Model_Entry (확정)  │ → 1차 진입 신호
                  │  80개 피처           │    (thr ≥ 0.25)
                  └──────────────────────┘
                  ┌──────────────────────┐
  M1 봉 데이터 ──→ │  Model_AddOn (개발중) │ → 2차/3차 추가 신호
  + 포지션 상태 ──→ │  80 + α 피처 (6개)   │
                  └──────────────────────┘
```

### Model_AddOn에 추가되는 α 피처 (6개)

| 피처 | 설명 |
|:---|:---|
| `unrealized_pnl_atr` | 현재 미실현 수익 / ATR (정규화) |
| `bars_since_entry` | 진입 후 경과 봉 수 |
| `bsp_scale_delta` | 진입 시점 대비 BSP Scale 변화 |
| `price_vs_entry` | (현재가 - 진입가) / ATR |
| `trend_acceleration` | BSP 기울기의 변화율 (추세 가속 중인가) |
| `addon_count` | 이미 추가한 횟수 (0, 1, 2) |

### 리스크 관리 (피라미딩 전용)

| 규칙 | 설명 |
|:---|:---|
| **최대 추가 횟수** | 2~3회로 제한 |
| **총 노출 한도** | 전체 포지션 합이 최대 리스크의 2배 미달 (Free Margin 기반) |
| **추가 진입 조건** | 그룹 포지션 전체가 **반드시 통합 수익 중**이어야 함 |
| **B/E 이동** | 추가 진입 즉시, 전체 그룹 포지션의 SL을 **평균 단가 + 총 마찰비용(30pt × n)** 이상으로 일괄 이동 |
| **일괄 청산** | TrailingStop 터치 시 그룹 내 **전체 포지션 동시 청산** |
| **Friction 누적** | 추가 진입마다 30pt × 진입 횟수만큼 마찰 비용 가중 |

## Phase 5: 피처 중요도 랏사이즈 + 소액 실전 + 에이전트 메모리 시스템

> **목표:** 전체 시스템을 통합하고 소액으로 실전 검증, 에이전트 메모리 시스템으로 적응적 학습

### 통합 시스템 전체 흐름

```
STEP 1: 현재 시간 확인 → 세션 필터 (런던/뉴욕만)
STEP 2: 현재 피처 벡터 생성 (핵심 피처 3~5개)
STEP 3: 벡터 DB에서 가장 일치하는 패턴 매칭 (0.01초)
STEP 4: 기대승률 > 60%? → 진입 결정
STEP 5: 피처 중요도 점수 계산 → 랏사이즈 결정
STEP 6: 진입 + Triple Barrier 설정
STEP 7: 자동 청산 (TP / SL / 시간 만료)
STEP 8: 거래 결과를 벡터 DB에 저장 → 에이전트 메모리 업데이트
```

### 에이전트 메모리 시스템 (FINMEM 스타일)

```
메모리 계층 구조:

  Layer 1 (단기 메모리): 최근 100거래 결과
    → 현재 전략의 실시간 성과 모니터링
    → "최근 10거래 중 7거래 손실" → 자동 경고

  Layer 2 (중기 메모리): 최근 1개월 국면별 성과
    → "금리 하락 국면에서 승률 72%, 상승 국면에서 45%"
    → 국면별 전략 가중치 자동 조정

  Layer 3 (장기 메모리): 전체 거래 이력의 패턴 DB
    → 과거 최적화 파라미터 이력
    → 성공/실패 거래의 공통 패턴
    → 구조적 변화(Structural Break) 이력
```

### 소액 실전 규칙

```
단계 1: 0.01 랏으로 1개월 (최소 100거래 이상)
단계 2: 0.03 랏으로 1개월
단계 3: 0.05 랏으로 1개월
     → 각 단계에서 Profit Factor > 1.2 유지 시만 다음 단계 진행

절대 규칙:
  • 한 거래 최대 손실 = 계좌의 2%
  • 일일 최대 손실 = 계좌의 5%
  • 연속 3회 손절 → 당일 거래 중단
  • 주간 최대 손실 = 계좌의 10% → 1주 일시 중단
```

## 전체를 관통하는 핵심 원칙

> [!IMPORTANT]
> **1. "얼마나 잘 맞추느냐"보다 "얼마나 안 잃느냐"가 중요**
> 진입 승률 55%라도 리스크 관리가 완벽하면 장기적으로 수익.
> 승률 70%라도 한 번의 대형 손실이 전부 날림.
>
> **2. Triple Barrier 라벨링(객관 정답지)은 절대로 건너뛰지 말 것**
> 인간의 주관적 라벨을 그대로 AI에 넣으면 GIGO(Garbage In, Garbage Out).
>
> **3. Walk-Forward 검증 없이 실전 투입하지 말 것**
> 2달→1년→10년 3단계 검증을 모두 통과한 패턴만 실전에 투입.
>
> **4. 베이스라인 먼저 측정할 것**
> 가장 단순한 규칙(예: `M5 BWMFI > 0`)의 승률을 먼저 구하고, AI 모델이 이보다 확실히 우수해야 합니다.

| # | 리스크 | 설명 | 대응 방안 |
|:---:|:---|:---|:---|
| 1 | **라벨 부족 → 과적합** | 수동 라벨 200개로 30+개 피처 학습 시 우연한 패턴을 진짜로 착각 | Triple Barrier로 전체 1분봉(수만 건)에 자동 라벨 확장 |
| 2 | **타임프레임 불일치** | M1 지표와 금리(일 단위)를 같은 Row에 넣으면 AI가 금리를 정적으로 오인 | 매크로 피처를 **변화율(Δ%)**이나 H1/H4 윈도우 평균으로 사전 변환 |
| 3 | **10년 데이터 스케일 왜곡** | 2016년 금값 $1,200 vs 2024년 $2,400 — 같은 ATR이라도 의미가 다름 | 모든 피처를 **멀티스케일 Z-score(60/240/1440) 또는 비율(%)**로 정규화 |
| 4 | **패턴 사전 노후화** | 3개월 전 패턴이 시장 구조 변화로 무효화 | 3개월 주기 정기 재마이닝 + 실시간 승률 모니터링(40% 이하 경고) |
| 5 | **거래 비용 미반영** | 스프레드·슬리피지를 라벨에 미반영 시 실전 손실 전환 | Triple Barrier 시 **IC Markets 스프레드 + 슬리피지** 차감 |
| 6 | **후행적 편향** | 과거 차트를 사후에 보며 라벨링하면 미래 정보 무의식 반영 | Triple Barrier 객관 채점 + Walk-Forward 3단계 기간 완전 분리 |
| 7 | **과도한 피처 수** | 100+개 피처를 넣으면 모델 수렴 불안정 + 잡음 패턴 학습 | SHAP/Permutation Importance로 **상위 10~15개만 유지** |
| 8 | **다중공선성** | 상관관계 높은 지표 쌍이 피처 중요도를 왜곡 | 상관계수 > 0.85인 피처 쌍 중 하나를 **사전 필터링** 제거 |
| 9 | **데이터 품질 오류** | 결측치·이상치가 벡터에 포함되면 거리 계산 왜곡 | ETL 파이프라인에 **정합성 검증 단계** 삽입 |
| 10 | **벡터 DB 조회 지연** | 실시간 매칭 시 0.1초 이상 걸리면 트레이딩 기회 상실 | 패턴 사전을 **10~20개 정예**로 유지 + HNSW 인덱스 최적화 |
| 11 | **생존자 편향** | 10년 백테스트 수익이 특정 1~2년 대박에 의해 왜곡 | **연도별 수익 분해** 확인 — 모든 연도에서 꾸준히 양수인지 검증 |
