> [!IMPORTANT]
> **🚨 피처 엔지니어링 3대 핵심 원칙 (절대 준수)**
> 1. **Shift+1 원칙**: 가이드에 나열된 모든 매크로 피처를 생성/맵핑할 때는 미래 데이터 오염(Look-ahead)을 막기 위해 무조건 Shift+1 한 값을 기준봉으로 삼는다.
> 2. **Friction Cost 30포인트**: 생성된 피처를 검증할 때(SHAP, 변수 중요도 등), 타겟 라벨(수익률)은 원시 수익률이 아닌 '30포인트 차감된 수익률'을 기준으로 하여 타당성을 판단한다.
> 3. **절대값 사용 금지**: 원본 절대값(가격 레벨 그 자체)은 트리기반 알고리즘의 비정상 분기를 초래하므로 아래 6가지 파생 스케일(Δ%, Z-Score 등)로 변환된 형태만 피처로 허용한다.

# XAUUSD AI 피처 완전 가이드

XAUUSD(금/달러) AI 트레이딩 모델에 활용할 수 있는 피처(Feature) 전체를 **중요도 순서**로 정리한 문서입니다.
IC Markets MT5에서 바로 사용 가능한 심볼명을 함께 표기했습니다.

> [!IMPORTANT]
> **2026-02-24 원칙 변경 — 메가 피처 풀(Mega Feature Pool) 도입:**
> 피처를 "하나씩 추가하며 테스트"하는 기존 방식 대신, **구할 수 있는 모든 피처(기술+매크로+세션)를 처음부터 전부 투입하고, AI(LightGBM + SHAP)가 핵심 피처를 알아서 골라내게** 합니다. AI는 수천 개의 차원을 동시에 처리하는 데 특화되어 있으므로, "100개 중 97개는 쓸모없고, 진짜 범인 3개만 발라냄"이 가능합니다.

## 🥇 1순위 — 반드시 포함해야 하는 핵심 피처

### ① 미국 10년물 국채 금리 변화 ✅ MT5 직접
**IC Markets 심볼: `UST10Y`**

금과 가장 직접적으로 연결된 변수입니다. 금은 이자를 주지 않는 자산이라, 국채 금리가 오르면 "차라리 채권 사서 이자를 받지" 하고 금에서 돈이 빠집니다. 반대로 금리가 떨어지면 금의 매력이 올라갑니다.

피처로 쓸 때는 금리의 **절대 수치(4.25%)가 아니라 "최근 며칠간 얼마나 변했는가"** 를 넣습니다. 금리가 빠르게 오르고 있는지, 내리고 있는지의 **방향과 속도**가 중요합니다.

- UST10Y 1일 변화율 → 단기 금리 충격
- UST10Y 5일 변화율 → 중기 금리 추세
- UST10Y 변화 가속도 → 금리가 빨라지는지 느려지는지

### ② 달러 강약 🧮 계산
**IC Markets 심볼: `EURUSD` (DXY 직접 심볼이 없을 경우 대용)**

금은 달러로 거래되기 때문에, 달러가 강해지면 금 가격이 내려가는 경향이 있습니다(역상관 약 -0.8). DXY의 57.6%가 유로화이므로 EURUSD를 뒤집으면(1/EURUSD) 달러 강약의 훌륭한 대용 지표가 됩니다.

> EURUSD 하락 = 달러 강세 = 금에 악재. 이 방향을 꼭 기억하세요.

보조로 `USDJPY`(달러/엔)도 사용합니다. 엔화가 약해질 때(USDJPY 상승) 위험회피 심리가 강해지며 금이 오르는 경우가 많습니다.

### ③ 금-DXY 롤링 상관계수 🧮 계산
XAUUSD와 EURUSD를 이용해 직접 계산합니다. 이 상관관계가 정상(-0.5 이하)이면 기존 패턴이 유효하고, -0.2 이상으로 올라오면 위기 국면으로 판단합니다.

| 상관계수 값 | 해석 | 거래 판단 |
|-----------|------|---------|
| -0.8 ~ -0.5 | 정상 (역상관 강함) | 기존 전략 유효 |
| -0.5 ~ -0.2 | 주의 (역상관 약화) | 포지션 크기 줄이기 |
| -0.2 이상 | 위험 (관계 붕괴) | 거래 중단 |

### ④ S&P 500 변화율 ✅ MT5 직접
**IC Markets 심볼: `US500`**

주식 시장이 급락하면 투자자들이 안전자산인 금으로 몰리는 경향이 있습니다. 단, 2020년 3월처럼 패닉 셀링 때는 금도 함께 빠질 수 있으므로(유동성 확보를 위해 모든 자산 매도), 주식 하락 = 금 상승으로 단순화하면 안 됩니다. 상관관계 피처와 함께 사용해야 합니다.

### ⑤ RSI 기울기 🧮 계산
일반적으로 RSI 절대값(35, 70 등)을 쓰지만, AI 피처로는 **"RSI가 얼마나 빠르게 변하는가(기울기)"** 가 더 유용합니다. RSI가 40에서 55로 급상승 중이면 모멘텀이 살아있다는 뜻입니다.

### ⑥ ATR 정규화 이격도 🧮 계산
현재 가격이 이동평균선에서 **ATR 기준으로 몇 배 떨어져 있는가**입니다. ATR로 나누면 변동성이 다른 시기에도 비교가 가능해집니다. (이평선과의 절대거리 $5는 의미 없지만, ATR의 2배 거리는 분명한 과열 신호입니다.)

### ⑦ 세션 구분 🧮 계산
거래 시간(KST)으로부터 계산합니다. 같은 기술적 신호라도 런던 세션에서 발생한 것과 아시아 세션에서 발생한 것의 성공 확률이 완전히 다릅니다.

| 세션 | 시간 (KST) | 특징 | 1분봉 적합도 |
|------|-----------|------|------------|
| 아시아 | 09:00~16:00 | 저변동, 횡보 | ❌ 부적합 (스프레드 비용 흡수 불가) |
| 런던 | 16:00~22:00 | 고변동, 추세 시작 | ⭐ 적합 |
| 뉴욕 | 22:00~07:00 | 뉴스 집중, 급변 | ⚠️ 이벤트 주의 |
| 런던+뉴욕 겹침 | 22:00~01:00 | 유동성 최대 | ⭐⭐ 최적 |

### ⑧ iBWMFI + CVD (M5 진입 신호) 🧩 계산 — Phase 1 핵심

> **2026-02-23 추가:** 1차 모델의 핵심 진입 규칙. 메타 라벨링(Phase 4)에서 AI가 "따를지 말지" 판단할 대상 신호입니다.

| 지표 | 역할 | AI 피처로 넣는 방법 |
|:---|:---|:---|
| **iBWMFI** (Bill Williams MFI) | 가격 움직임의 효율성 측정 | 절대값보다 **부호(양수/음수)**와 **변화율** 사용 |
| **CVD** (Cumulative Volume Delta) | 매수/매도 압력 방향 | **전환점(Cross zero)** 및 **기울기** 사용 |

```
M5 진입 조건 예시 (매수):
  iBWMFI > 0        → 시장 참여도 상승 중
  CVD 상승 전환     → 매수 압력이 매도 압력을 초과
  → 두 조건 동시 충족 시 진입 신호 발생
```

이 지표들은 **순수 기술적 지표**(가격+거래량 기반)이므로 Phase 1에서 즉시 사용합니다. 매크로 피처(Phase 3)에 넣지 않습니다.

### ⑨ H4 ADX + iBWMFI (추세 필터) � 계산 — Phase 1 단순 → Phase 3 고급화

> **2026-02-23 추가:** 상위 타임프레임 추세 필터로, Phase 1에서 단순 필터로 시작하고 Phase 3에서 매크로 피처와 결합하여 고급화

| 사용 Phase | H4 활용 방법 |
|:---|:---|
| **Phase 1** | 단순 필터: `H4 ADX > 20` + `iBWMFI > 0` 이면 진입 허용, 아니면 대기 |
| **Phase 3** | 매크로 국면(GO/NO_GO)과 결합: 방향 일치 시 풀 사이즈, 불일치 시 50% 축소 |

**왜 매크로 피처가 아닌가?**
- ADX와 iBWMFI는 **가격/거래량 기반 순수 기술적 지표**입니다
- 매크로 피처는 UST10Y(금리), EURUSD(달러), US500(주식) 같은 **외부 거시경제 데이터**를 말합니다
- H4 지표는 **기술적 국면 필터**(Layer 1.5)이지, 매크로 국면 필터(Layer 1)가 아닙니다

## �🥈 2순위 — 추가하면 성능이 크게 향상되는 피처

### ① 실질금리 변화 🌐 외부 API
**데이터 소스: FRED API (`DFII10` — 미국 10년물 TIPS)**

금과 가장 강한 역상관관계(-0.8)를 가지는 변수입니다. **명목금리(UST10Y)에서 기대인플레이션을 뺀 것**으로, "인플레이션을 감안해도 채권이 돈이 되느냐"를 보여줍니다.

- 실질금리 상승 → 현금/채권의 실질 수익 상승 → 금 매력 감소
- 실질금리 하락 → 현금 보유 비용 증가 → 금 선호

FRED는 무료 API를 제공합니다. 일봉 데이터를 매일 자동으로 받아 1분봉에 forward-fill로 매핑하면 됩니다.

### ② 장단기 금리차 🧮 계산
**IC Markets 심볼: `UST10Y` - `UST05Y`**

두 심볼 모두 IC Markets MT5에서 제공합니다. 10년물과 5년물 금리 차를 계산합니다(원래는 2년물이 이상적이나 IC Markets이 UST05Y 제공).

이 값이 마이너스(역전)가 되면 시장은 **경기침체**를 예상하고 있다는 뜻 → 안전자산 선호 → 금 강세 경향. 값 자체보다 **"좁혀지는 속도"** 가 더 중요한 피처입니다.

### ③ VIX (공포지수) 🌐 외부 API
**데이터 소스: FRED API (`VIXCLS`) 또는 Yahoo Finance (`^VIX`)**

IC Markets에서 직접 제공하지 않습니다. 시장의 공포 심리를 수치화한 것으로, VIX가 20 이하면 안정적, 30 이상이면 불안한 상태입니다.

- VIX 급등 → 공포 확산 → 금 상승 (안전자산 매수)
- VIX 안정 → 위험자산 선호 → 금 횡보/하락

**VIX의 변화 속도(하루 5포인트 이상 급등)**가 특히 중요한 피처입니다.

### ④ 원유 변화율 ✅ MT5 직접
**IC Markets 심볼: `XTIUSD` (WTI), `XBRUSD` (브렌트)**

원유 가격 급등 → 인플레이션 기대 상승 → 실질금리 하락 → 금 상승 경로가 있습니다. 유가 급등은 지정학적 불안의 신호이기도 해서 금에 이중으로 영향을 줍니다.

### ⑤ 은(Silver) 관련 비율 ✅ MT5 직접
**IC Markets 심볼: `XAGUSD`**

금/은 비율(XAUUSD ÷ XAGUSD)이 80 이상이면 금이 상대적으로 비싸다는 뜻입니다. 역사적으로 이 비율이 극단적으로 높아지면 이후 금이 조정받거나 은이 따라잡는 경향이 있습니다. 귀금속 섹터 전체의 흐름을 파악하는 데에도 유용합니다.

### ⑥ 볼린저%b + 밴드 폭 변화 🧮 계산
단순히 "밴드 상단/하단 터치"가 아니라, **%b(밴드 내 위치)** 와 **밴드 폭(수축/확장 속도)** 을 함께 넣습니다. 밴드가 좁아지고 있으면 곧 큰 변동이 온다는 신호이고, 이 상태에서 %b가 어떤 방향으로 움직이는지가 중요합니다.

### ⑦ 금리 사이클 국면 🧮 계산
연준(Fed)이 금리를 인상 사이클인지 인하 사이클인지에 따라 금의 큰 방향이 달라집니다. UST10Y의 20일 변화율로 국면을 분류합니다:
- **HIKING** (인상 중): 금에 불리 → 신호 강도 낮춤
- **CUTTING** (인하 중): 금에 유리 → 신호 강도 높임
- **STABLE** (안정): 다른 요인이 주도

## 🥉 3순위 — 고급 단계에서 추가하는 피처

### ① 구리 가격 비율 🌐 외부 API
**데이터 소스: Yahoo Finance (`HG=F`)**

IC Markets에서 직접 제공하지 않습니다. 구리는 "닥터 코퍼"라 불리며 경기 선행 지표로 쓰입니다. **금/구리 비율(Gold/Copper Ratio)** 이 높아지면 시장이 "방어적"이 되고 있다는 뜻으로, 경기 침체 우려 강화 신호입니다.

### ② 금-금리 롤링 상관계수 🧮 계산
XAUUSD와 UST10Y 사이의 20일 롤링 상관계수입니다. 이 값이 정상(-0.5~-0.7)이면 금리가 금을 설명하고 있음을 의미하고, 관계가 깨지면 지정학, 중앙은행 매입 등 다른 힘이 작동 중이라는 신호입니다.

### ③ 계절성 (월별/요일) 🧮 계산
현재 날짜에서 계산합니다.

**강세 시기 (역사적 통계 기반):**
- 1~2월: 인도/중국 설 수요, 연초 포트폴리오 조정
- 7~9월: 인도 결혼/축제 시즌 수요
- 11~12월: 연말 안전자산 수요

**약세 시기:**
- 3~4월: 세금 시즌 현금화
- 6월: 상반기 포트폴리오 리밸런싱

**요일별:** 금요일 오후(변동성 감소), 수요일(FOMC 영향 높음).

### ④ 경제 서프라이즈 지수 🌐 외부 API (유료)
경제 지표가 시장 예상 대비 좋게/나쁘게 나왔는지를 종합한 지수입니다. 블룸버그 구독이 필요해 개인 트레이더에게는 접근이 어렵습니다. 대안으로 미국 주요 경제 지표(NFP, CPI, PCE)의 실제치 vs 예상치 차이를 직접 계산해서 사용할 수 있습니다.

### ⑤ 중앙은행 금 순매입 🌐 외부 (분기별)

## 🧩 메타 피처 — 관계 자체를 피처로 (전 단계에서 가장 고급)

개별 피처 값이 아니라, **피처들 간의 관계 자체**를 넣는 고급 기법입니다.

| 메타 피처 | 계산 방법 | 의미 | 활용 |
|----------|---------|------|------|
| 금-달러 상관계수 | XAUUSD vs EURUSD (20일 롤링) | "달러와 금의 관계가 정상인가?" | 붕괴 시 거래 중단 |
| 금-금리 상관계수 | XAUUSD vs UST10Y (20일 롤링) | "금리가 금을 설명하는가?" | 관계 약화 = 다른 힘 작동 중 |
| 금-주식 상관계수 | XAUUSD vs US500 (20일 롤링) | "공포가 금을 밀어올리는가?" | 공포 vs 인플레 국면 구분 |
| 금-원유 상관계수 | XAUUSD vs XTIUSD (20일 롤링) | "인플레 경로로 금이 움직이는가?" | 국면 분류에 활용 |

> [!WARNING]
> 이 상관관계들이 **동시에 붕괴되면** → 시장 구조 자체가 변하고 있다는 뜻 → 모든 거래를 중단해야 하는 "구조적 위기 국면"입니다. 2020년 코로나 초기, 2022년 금리 급등기가 해당됩니다.

## 💰 응용: 피처 중요도 → 랏사이즈 자동 결정

> **"AI는 방향을 결정하고, 피처 중요도 가중 점수는 그 결정에 얼마나 베팅할지를 결정한다."**

피처들이 얼마나 **같은 방향을 일치되게 가리키는가**에 따라 랏사이즈를 동적으로 조절하는 방법입니다.

### 동작 원리

```
피처들이 얼마나 "같은 방향"을 가리키는가?
           ↓
    종합 신호 강도(점수) 계산
           ↓
     강도에 따라 랏사이즈 결정
```

### ① 각 피처 → 방향 우호도 점수화 (매수 예시)

| 피처 | 현재 상태 | 가중치 | 점수 |
|------|----------|-------|------|
| 실질금리 (DFII10) | 하락 중 | 높음 (AI 중요도 25%) | ✅ +2.5점 |
| UST10Y 변화율 | 5일간 0.15% 하락 | 높음 | ✅ +2.0점 |
| EURUSD | 0.5% 상승 (달러 약세) | 높음 | ✅ +2.0점 |
| US500 변화율 | 1.2% 하락 (안전자산 수요) | 중간 | ✅ +1.5점 |
| VIX | 18 → 23 상승 (공포 증가) | 중간 | ✅ +1.5점 |
| 금-달러 상관계수 | -0.72 (정상) | 낮음 | ✅ +0.5점 |
| RSI 기울기 | 하락 후 반등 중 | 낮음 | ✅ +0.5점 |
| 세션 | 런던+뉴욕 겹침 | 낮음 | ✅ +0.5점 |
| **합계** | | | **11/13점 (85%)** |

> 각 피처의 가중치는 AI 모델 학습 후 나오는 **피처 중요도(Feature Importance)** 를 그대로 반영합니다.

### ② 종합 점수 → 랏사이즈 매핑

| 종합 점수 | 의미 | 랏사이즈 | 예시 (기본 0.1 랏 기준) |
|---------|------|---------|---------------------|
| 80% 이상 | 거의 모든 신호 일치 | **풀 사이즈** | 0.10 랏 |
| 60~80% | 대부분 일치, 일부 불확실 | **하프 사이즈** | 0.05 랏 |
| 40~60% | 신호 혼재 | **쿼터 사이즈** | 0.025 랏 |
| 40% 미만 | 신호 불일치 | **거래 안 함** | 0 랏 |

→ 위 예시(85%) → **풀 사이즈(0.1 랏)** 로 진입

### ③ 왜 이 방법이 좋은가?

**기존 방법 (고정 랏):**
- AI가 "매수" 신호 → 항상 0.1 랏 진입
- AI가 틀리면 항상 같은 크기의 손실

**피처 중요도 기반 사이징:**
- AI의 확신도가 낮을 때 → 작은 랏 → 손실 제한
- AI의 확신도가 높을 때 → 큰 랏 → 수익 극대화
- **"틀려도 조금 잃고, 맞으면 크게 버는"** 구조가 자동으로 만들어짐

### ④ 반드시 지켜야 할 안전 규칙

> [!CAUTION]
> 점수가 아무리 높아도 아래 규칙은 무조건 우선합니다.

1. **상관관계 붕괴 시 → 랏사이즈 강제 0**: 금-달러 상관계수가 -0.2 이상이면(관계 붕괴) 점수와 무관하게 거래 중단
2. **연속 손절 시 → 자동 축소**: 3연패 이후에는 점수와 무관하게 최소 랏(0.01)으로 강제 전환
3. **절대 한도**: 어떤 경우에도 한 거래에서 계좌의 **2%를 초과하는 손실 위험**은 노출하지 않음
4. **뉴스 이벤트 시 → 랏사이즈 0**: FOMC, NFP, CPI 발표 전후 30분은 강제 거래 중단

### ⑤ 이 방법과 Layer 1/2/3의 관계

```
Layer 1 (국면 필터)   → GO / NO_GO 결정
         ↓ GO
Layer 2 (진입 신호)   → BUY / SELL / WAIT 결정
         ↓ BUY/SELL
피처 중요도 점수 계산 → 종합 신호 강도 산출
         ↓
랏사이즈 결정         → 0 / 0.025 / 0.05 / 0.1 랏
         ↓
Layer 3 (청산 배리어) → TP / SL / 시간 만료
```

피처 중요도 기반 랏사이즈 결정은 Layer 2와 Layer 3 **사이에서 동작**하는 독립적인 모듈입니다.

## 📐 1단계 피처별 계산 타임프레임 & 분석 방법

> 거래는 1분봉(M1)으로 하지만, 피처는 역할에 따라 **다른 타임프레임**으로 계산하고 M1에 "붙여" 사용합니다.

### 전체 원리 — "위에서 아래로 흘러내림"

```
H1봉 (1시간봉)  → 매크로/국면 피처 계산
      ↓ 미래 참조 편향 방지: 직전 완성봉(Shift +1) 데이터만 사용!
      ↓ forward-fill (H1값을 M1에 복사)
M1봉 (1분봉)   → 기술적 진입 피처 계산
      ↓ 합치기
   최종 피처셋 → AI 모델 입력
```

> [!CAUTION]
> **미래 참조 편향(Look-ahead Bias) 방지 필수:**
> 09:00분 H1 봉의 종가는 09:59:59에 확정됩니다. 만약 09:15분 M1 봉에 09:00분 H1 봉 데이터를 붙이면, AI가 45분 뒤의 미래를 미리 보게 되어 백테스트 수익률이 비정상적으로 높게 나옵니다.
> **반드시 상위 타임프레임(H1) 매핑 시 `직전 완성봉(Shift +1)`의 데이터만을 M1 차트에 결합**해야 합니다. (예: 09:15 M1 봉에는 08:00 H1 봉의 완성 데이터를 붙임)

### 각 피처별 계산 타임프레임

#### ① UST10Y — H1봉으로 계산

| 피처 | 계산 방법 | 왜 H1인가? |
|------|---------|----------|
| 1일 변화율 | 현재 H1 - 24봉 전 H1 | 금리는 분 단위로 의미있게 변하지 않음 |
| 5일 변화율 | 현재 H1 - 120봉 전 H1 | 추세 방향 파악에는 H1로 충분 |
| 변화 가속도 | (1일 변화율) - (어제 1일 변화율) | 금리가 빨라지는지 느려지는지 |

> UST10Y를 1분봉으로 보면 대부분 변화가 없거나 노이즈만 있습니다. H1이 의미있는 단위입니다.

#### ② EURUSD — H1봉으로 계산

| 피처 | 계산 방법 |
|------|---------|
| 달러 변화율 (1일) | 현재 H1 종가 - 24봉 전 H1 종가 |
| 달러 변화율 (5일) | 현재 H1 종가 - 120봉 전 H1 종가 |
| 달러 방향 | H1 EMA(20) 기울기 (상승 = 달러 약세 = 금 호재) |

#### ③ US500 — H1봉으로 계산

| 피처 | 계산 방법 |
|------|---------|
| 일간 변화율 | 현재 H1 - 어제 장 마감 H1 |
| 위험선호 방향 | H1 EMA(20) 기울기 (하락 = 위험회피 = 금 호재) |

#### ④ XAGUSD — 현재 가격으로 즉시 계산

| 피처 | 계산 방법 |
|------|---------|
| 금/은 비율 | 현재 XAUUSD ÷ 현재 XAGUSD |
| 비율 변화 | 현재 비율 - 24봉 전 비율 (H1 기준) |

#### ⑤ XTIUSD — H1봉으로 계산

| 피처 | 계산 방법 |
|------|---------|
| 원유 변화율 (1일) | H1 기준 24봉 변화율 |
| 원유 방향 | H1 EMA(10) 기울기 |

#### ⑥ 세션 구분 — 현재 시간으로 즉시 계산

```
현재 KST 시간 → 아시아(0점) / 런던(2점) / 뉴욕(1점) / 겹침(3점)
계산 자체가 필요 없음 — 시계만 보면 됨
```

#### ⑦ XAUUSD vs EURUSD 상관계수 — H1 480봉으로 계산

가장 많은 데이터가 필요합니다:

```
H1 480봉 = 24시간 × 20일 = 20거래일치 데이터
XAUUSD H1 종가 480개  ←─┐
EURUSD H1 종가 480개  ←─┘ → 피어슨 상관계수 계산
```

### M1 진입 시점에서 실제 동작 흐름

매 1분봉이 완성될 때 다음을 계산합니다:

```
[매크로 피처 — H1에서 계산 후 M1에 복사]
  UST10Y H1 24봉 변화율    = -0.12% (금리 하락 중 ✅)
  EURUSD H1 24봉 변화율    = +0.4%  (달러 약세 ✅)
  US500  H1 24봉 변화율    = -0.8%  (주식 하락, 안전자산 ✅)
  XAUUSD/XAGUSD 비율 변화  = +0.5   (금이 상대적 강세)
  XTIUSD H1 변화율         = +1.2%  (유가 상승, 인플레 ✅)
  상관계수 H1 480봉         = -0.71  (정상 ✅)
  세션                      = 런던+뉴욕 겹침 ✅

[기술적 피처 — M1에서 계산]
  RSI(14) 기울기   = 상승 중
  ATR 이격도       = 정규화된 이평선 거리
  볼린저 %b        = 0.65

[최종 결합]
  → 매크로 7개 + 기술적 3개 = 총 10개 피처 → AI 모델 입력
```

### 데이터 필요량 요약

| 타임프레임 | 필요 봉 수 | 용도 |
|-----------|----------|------|
| **H1 480봉** | 20일치 | 상관계수 계산 (가장 많음) |
| H1 120봉 | 5일치 | UST10Y, EURUSD 5일 변화율 |
| H1 24봉 | 1일치 | 나머지 매크로 1일 변화율 |
| M1 30봉 | 30분치 | RSI, ATR, 볼린저 |

> **결론:** 매크로 피처는 **H1봉 24~480봉**으로 한 번 계산하고 M1에 복사해서 쓰고, 기술적 피처는 **M1봉 30봉**으로 실시간 계산합니다. 두 종류를 합쳐서 AI에 넣는 것이 1단계 피처셋입니다.

## ⚖️ 피처 스케일링(정규화) 가이드 가이드라인

AI 모델(특히 신경망, 선형 모델 계열)은 입력 데이터의 크기(Scale)와 분포에 매우 민감합니다. 금리 변화율(-0.15%), S&P 잔차, RSI(35) 등 단위가 제각각인 피처들을 그대로 넣으면 AI가 제대로 학습하지 못합니다.

- **정적 MinMax 남용 금지:** 시계열 데이터 특성상 과거의 고점/저점이 새롭게 갱신될 수 있으므로 고정된 최대/최소값을 사용하는 MinMax 스케일링은 지양합니다.
- **권장 ①: 롤링 Z-스코어 (Rolling Z-Score)**
  - `(현재값 - 최근 N주기 평균) / 최근 N주기 표준편차`
  - 예: 최근 120주기 단위의 롤링 Z-스코어를 사용하면 언제나 일관된 분포(-3 ~ +3 등)를 갖게 됩니다.
- **권장 ②: 랭크 기반 스케일링 (Rank-based Scaling)**
  - 최근 N주기 내에서 현재 값이 상위 몇 %에 위치하는지 백분위 등수(0~1)로 변환합니다.
  - 변동성이 극심하게 튀는 이상치(Outlier)에 강한 저항력을 갖습니다.

각 피처의 성격에 맞게 위 두 가지 스케일링 기법 중 하나를 적용하여 모델에 입력하는 전처리 레이어(Preprocessing Layer) 도입이 반드시 필요합니다.

## 🎯 진짜 남은 숙제 — 정확한 진입 타이밍

지금까지 정리한 것은 큰 그림(국면 판단, 피처, 랏사이즈, 청산)이고, **실질적인 진입 시점 결정**은 여전히 핵심 숙제입니다.

### 지금까지 완성한 것 vs 남은 것

```
✅ 완성한 것 (큰 그림):
  Layer 1: 국면 판단 → "거래 가능 환경인가?"
  피처 가이드       → "무엇을 AI에 넣을 것인가?"
  랏사이즈 결정     → "얼마나 베팅할 것인가?"
  Layer 3: 청산     → "어디서 나올 것인가?"

❓ 아직 풀지 못한 것 (핵심):
  Layer 2: 진입     → "바로 이 1분봉에서 들어가는가?"
```

### 진입 타이밍이 어려운 이유

솔직히 말하면, **이 부분은 AI도 잘 못합니다.**

XAUUSD의 **신호 대 잡음 비율이 3~8%**밖에 안 됩니다. 즉 100번의 1분봉 중에서 "확실한 진입점"은 3~8개 정도이고, 나머지 92~97개는 노이즈입니다.

---

### 현실적으로 가능한 3가지 접근법

#### 접근법 A: "예측하지 말고 반응하라" ⭐ 권장 (시작점)

> 정확한 진입점을 AI로 예측하는 대신, 신호가 나왔을 때 **"따를 것인지 말 것인지"만 판단**합니다.

```
기존 규칙 기반 신호 (EMA 크로스, RSI 반등 등)
           ↓ 신호 발생
AI 메타 라벨: "이 신호를 지금 따를 가치가 있는가?"
           ↓ Yes
피처 중요도 → 랏사이즈 결정 후 진입
```

- AI의 역할: 예측이 아니라 **필터링**
- 승률 자체는 50~55%로 낮지만, 좋은 신호만 따르면 실질 승률이 올라감
- **이것이 "메타 라벨링"의 본질이고, 실전에서 가장 성공적인 방법**

#### 접근법 B: "확률 분포를 예측하라" (중급)

> AI에게 "지금 매수야 매도야?"를 묻지 말고, **"30분 안에 +$1.2를 먼저 터치할 확률이 몇 %인가?"** 를 묻기

```
AI 출력: "TP 먼저 도달 확률 = 63%"
         → 60% 이상이면 진입
         → 70% 이상이면 풀사이즈 진입
```

- Triple Barrier 라벨로 학습시킨 모델의 `predict_proba()`가 바로 이것
- 방향이 아니라 **확률**로 진입 여부를 결정

#### 접근법 C: "강화학습(RL)으로 AI가 스스로 배우게" (고급)

> 진입/관망/청산을 AI가 시행착오를 통해 직접 학습

- 가장 이론적으로 우아하지만, **구현 난이도가 극히 높고 학습 시간이 매우 김**
- A → B를 거쳐 경험을 쌓은 뒤에야 시도할 수 있는 단계

### 💡 핵심 결론

> [!IMPORTANT]
> **"정확한 진입 시점"을 완벽하게 맞추는 것은 AI조차 불가능합니다.**
>
> 프로 퀀트들은 진입 자체에 집착하지 않고:
> 1. **들어가도 되는 환경**을 엄격히 통제하고 (Layer 1 ✅)
> 2. **대충 진입해도 손해가 제한**되게 랏사이즈를 조절하고 (랏사이즈 ✅)
> 3. **나오는 규칙을 기계적으로** 지키는 것 (Layer 3 ✅)
>
> 에 집중합니다. 진입 정확도가 55%만 되어도, 위 3가지가 완벽하면 장기적으로 수익이 납니다.
>
> **"어디서 들어가느냐"보다 "얼마나 잃지 않느냐"가 진짜 게임입니다.**

### 현실적인 다음 단계

**접근법 A(메타 라벨링)부터 시작**하는 것을 권장합니다.
기존에 사용하시는 인디케이터 조합을 **1차 신호**로 쓰고, AI는 **2차 필터**로만 활용하는 방식입니다.

## 🧩 메타 피처(Meta Feature)의 정확한 의미

**"피처에 대한 피처"** 입니다. 즉, 원래 데이터 값이 아니라 **피처들 사이의 관계나 상태** 자체를 새로운 피처로 만드는 것입니다.

### 일반 피처 vs 메타 피처

| 구분 | 예시 | 의미 |
|------|------|------|
| **일반 피처** | EURUSD = 1.0850 | 달러/유로의 현재 가격 |
| **일반 피처** | UST10Y = 4.25% | 금리의 현재 값 |
| **메타 피처** | EURUSD와 XAUUSD의 20일 상관계수 = -0.71 | 달러와 금의 **관계**가 얼마나 강한가 |

### 쉬운 비유

> 일반 피처가 **"지금 온도는 몇 도?"** 라면,
> 메타 피처는 **"오늘과 어제의 온도 관계가 평소와 다른가?"** 입니다.

개별 값이 아니라 **"값들이 서로 어떻게 반응하고 있는가"** 를 측정합니다.

### 왜 메타 피처가 중요한가?

금과 달러의 일반 피처(각각의 가격)는 항상 존재합니다. 하지만 이 둘이 **"정상적인 역상관 관계를 유지하고 있는가?"** 는 별도로 계산해야 알 수 있습니다.

```
정상:  달러 ↑  →  금 ↓   (상관계수 -0.7)
위기:  달러 ↑  →  금 ↑   (상관계수 +0.3) ← 관계가 깨짐!
```

이 "관계가 깨졌다"는 사실 자체를 AI에게 알려주는 것이 메타 피처입니다.

### 메타 피처 종류 (우리 전략 기준)

| 메타 피처 | 무엇의 관계인가? | 알려주는 것 |
|----------|--------------|-----------|
| 금-달러 롤링 상관계수 | XAUUSD ↔ EURUSD | 기존 역상관 패턴이 살아있는가? |
| 금-금리 롤링 상관계수 | XAUUSD ↔ UST10Y | 금리가 금을 설명하고 있는가? |
| 금-주식 롤링 상관계수 | XAUUSD ↔ US500 | 공포 심리가 금을 움직이는가? |
| 금/은 비율 변화 | XAUUSD ÷ XAGUSD | 귀금속 섹터 내 금의 상대적 위치 |

> [!NOTE]
> 메타 피처는 "시장이 지금 어떤 논리로 움직이고 있는가"를 AI에게 알려주는 **"시장 구조 감지기"** 입니다. 개별 가격 값만 보는 AI는 관계가 깨지는 위기 국면에서 대형 손실을 냅니다. 메타 피처가 있으면 AI가 "지금은 평소 패턴이 통하지 않는 국면"임을 인식할 수 있습니다.

## 📖 Triple Barrier (트리플 배리어)의 정확한 의미

**"3개의 벽"** 이라는 뜻입니다. 진입한 순간, 가격 차트 위에 **3개의 경계선(벽)**을 그려놓고, 가격이 **어떤 벽을 먼저 터치하느냐**에 따라 결과를 결정하는 라벨링 방법입니다.

### 3개의 벽

```
가격
  ↑
  │   ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄  ① 상단 배리어 (이익 실현선, TP)
  │
  │         📈
  │       ╱    ╲
  │     ╱        ╲     ← 실제 가격 움직임
  │   ╱            ╲ 📉
  ●─────────────────── ← 진입 가격
  │
  │   ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄  ② 하단 배리어 (손절선, SL)
  │
  └─────────────────────────→ 시간
       ③ 세로 배리어 (시간 만료)
       예: 30분 후 │
```

| 벽 | 이름 | 의미 | 비유 |
|----|------|------|------|
| ① 상단 | Take Profit (TP) | 가격이 여기까지 올라가면 → **성공** | 천장 |
| ② 하단 | Stop Loss (SL) | 가격이 여기까지 내려가면 → **실패** | 바닥 |
| ③ 세로 | Time Limit | 시간 내에 TP도 SL도 안 건드리면 → **무효** | 마감시간 |

### XAUUSD 1분봉 실제 예시

```
진입가: $2,850.00
ATR(14): $0.80

① 상단 배리어: $2,850.00 + ($0.80 × 1.5) = $2,851.20  (+$1.20)
② 하단 배리어: $2,850.00 - ($0.80 × 1.0) = $2,849.20  (-$0.80)
③ 세로 배리어: 30분 후

→ 30분 안에 $2,851.20을 먼저 터치하면: 라벨 = 1 (성공)
→ 30분 안에 $2,849.20을 먼저 터치하면: 라벨 = 0 (실패)
→ 30분 지나도 둘 다 안 터치하면:       라벨 = 0 (무효)
```

### 왜 Triple Barrier를 쓰는가?

**기존 라벨링 방식의 문제:**
"앞으로 가격이 오를 것인가?" → $2,850 에서 $2,870까지 올랐으니 "매수 성공"이라고 라벨링. 하지만 중간에 $2,840까지 -$10 빠졌다가 올라감 → **실전에서는 손절 당했을 수도 있음**

**Triple Barrier의 해결:**
"정해진 손절/이익/시간 안에서 수익이 나는가?" → 최대 -$0.80까지 버틸 수 있는 상태에서 +$1.20이 먼저 나오는가를 판정 → **실전 거래와 동일한 조건**으로 라벨링하기 때문에 훨씬 현실적

### 한 줄 정리

> 기존 라벨링이 "미래를 알고 나서 판단"한 것이라면, Triple Barrier는 **"실전처럼 리스크 범위를 설정하고 그 안에서 판단"** 하는 것입니다.

이 개념은 퀀트 금융의 대표 교과서 **Marcos López de Prado**의 *Advances in Financial Machine Learning*에서 제안된 방법론입니다.

#### 🔹 수익률 곡선 국면
5년·10년·30년물 채권 3종으로 **수익률 곡선의 형태**를 분류합니다. 곡선이 **역전**(5Y > 10Y > 30Y)되면 경기 침체 신호 → 금리 인하 기대 → 금에 강한 호재. 곡선이 **스티프닝**(장단기차 확대 중)이면 인플레 기대 상승 → 역시 금에 호재. 반대로 **정상 정배열**이면 경기 회복기 → 금리 상승 압력 → 금에 불리. 이 3가지 피처(장단기차, 곡률, 변화 속도)를 국면 판단 모델에 추가합니다.

#### 🔹 크로스 귀금속 모멘텀
금·은·플래티넘·팔라듐 4종의 **5일 수익률 순위**를 매깁니다. 금이 1위(가장 강한 상승)면 모멘텀 최강 → 추세 추종 매수 적극화. 금이 4위면 귀금속 안에서도 약한 것이므로 매수 회피. 특히 은이 금보다 급등하면(금은비 하락) 투기적 과열 신호로, 반전 위험이 높아집니다.

#### 🔹 멀티 세션 컨센서스
단일 세션 필터를 넘어 **연속 세션 간 방향 일치도**를 점수화합니다. 아시아 세션에서 금이 상승 방향이었고, 런던 세션 시작 30분 후에도 상승 방향이 유지되면 **컨센서스 +2점**. 여기에 해당 세션의 벤치마크 지수(아시아→JP225, 런던→DE40)도 같은 방향이면 +3점. 세션 간 합의가 강할수록 큰 포지션, 갈등(엇갈림) 시 관망합니다.

#### 🔹 VIX 유동 손절
`VIX_H6` 값에 따라 **TP/SL 배리어 폭을 자동 조절**합니다. VIX < 15(저변동기)에는 TP=ATR×1.0, SL=ATR×0.8로 타이트하게. VIX 25~35(공포)에는 TP=ATR×2.5, SL=ATR×1.5로 넓혀서 "큰 파도"를 타게 합니다. VIX > 35(패닉)에는 유동성 소멸 위험으로 포지션 50% 감축 또는 거래 중단. 같은 전략이라도 시장 변동성에 맞춰 배리어가 숨을 쉬게 됩니다.

## 🛠️ 피처 엔지니어링 실전 가이드

피처를 "무엇을 넣을까"만큼 중요한 것이 **"어떻게 넣고, 어떻게 검증하고, 어떻게 관리 하는가"** 입니다. 아래는 실전 모델 개발 시 반드시 지켜야 하는 원칙들입니다.

### ① 피처 간 공선성(Multicollinearity) 검증

`UST10Y_H6 변화율`과 `EURUSD 변화율`은 서로 강하게 연동됩니다 (미국 금리 상승 → 달러 강세 → EURUSD 하락). 같은 정보를 두 번 넣으면 AI가 피처 중요도를 왜곡합니다.

**해결책:** 1순위 피처로 모델을 한 번 돌린 뒤, 피처 간 **상관행렬(Correlation Matrix)**을 반드시 뽑아보세요. 서로 **0.7 이상의 상관관계**를 가진 피처 쌍이 있으면, 하나를 제거하거나 둘을 합쳐 하나의 "복합 달러 압력 지수"로 만드는 것이 성능을 더 높입니다.

```python
# 공선성 확인 예시
import pandas as pd
features_df = pd.DataFrame({
    'UST10Y_chg': ust10y_change,
    'EURUSD_chg': eurusd_change,
    'US500_chg': us500_change
})
print(features_df.corr())
# UST10Y_chg와 EURUSD_chg가 0.75 이상이면 → 하나로 합치기 고려
```

### ② 피처 선택 방법 — SHAP 자동 추출 + Ablation 검증

> **2026-02-24 업데이트:** 기존 "1개씩 추가" 방식에서 **"전부 넣고 AI가 핵심만 추출"** 방식으로 변경. Ablation은 새 지표 유효성 확인 목적의 오프라인 연구용으로 보존.

**1단계: 메가 피처 풀 → SHAP 자동 추출 (주력)**
```
모든 피처(기술+매크로+세션) 50~100개를 통째로 LightGBM에 투입
      ↓
SHAP Feature Importance 리포트 생성
      ↓
"승률 75%를 만든 1등 공신은 [BWMFI] 35%, 2등은 [UST10Y Δ%] 20%"
      ↓
상위 10~15개만 유지, 하위 80% 제거
→ SHAP이 자동으로 "진짜 범인"을 발라냄
```

**2단계: Ablation Study — 새 지표 검증용 (보조)**

새로 개발한 지표(예: BWMFI 개량형)가 기존 모델에 실제로 기여하는지 확인할 때만 사용합니다.

```
실험 1: SHAP 상위 10개 피처만    → 승률 58%, Sharpe 1.1
실험 2: + 새 지표 1개 추가        → 승률 60%, Sharpe 1.2 ✅ 효과 있음
실험 3: + 또 다른 새 지표 추가    → 승률 60%, Sharpe 1.15 ❌ 미미, 제거
```

> [!NOTE]
> **과도한 피처 수 리스크:** 100+개 피처를 그대로 넣으면 모델 수렴이 불안정해지고 잡음 패턴을 학습합니다. SHAP/Permutation Importance로 **상위 10~15개만 유지**하는 것이 핵심입니다.

### ③ 파생 피처(Derived Feature) 가공 공식 레퍼런스

모든 피처는 아래 6가지 유형 중 **최소 하나로 가공**되어야 합니다. "원본값 그 자체"를 넣는 피처는 하나도 없어야 합니다.

> [!CAUTION]
> **원본 가격(Close Price)은 절대로 넣지 마세요.** `XAUUSD_Close = 2854.12` 같은 값 자체는 AI에게 넣으면 안 됩니다. 과거 $1,700~$2,000 범위에서 학습한 AI는 $2,800을 보는 순간 패닉에 빠져 무한 숏(매도)만 치다가 계좌를 박살냅니다. **"절대 좌표"가 아니라 "상대적 나침반(변화율, 이격도, Z-Score)"** 만 먹여야 시대가 변해도 AI가 동일한 지능을 발휘합니다.

| 유형 | 공식 | 의미 | 적용 예 |
|------|------|------|---------|
| **변화율 (Return)** | `(현재값 - n봉 전 값) / n봉 전 값` | 속도 | UST10Y 1일/5일 변화율 |
| **가속도 (Acceleration)** | `현재 변화율 - 직전 변화율` | 속도의 변화 | RSI가 빨라지는 중인가? |
| **Z-Score** | `(현재값 - 120봉 평균) / 120봉 표준편차` | 극단값 탐지 | VIX가 "비정상적으로" 높은가? |
| **정규화 이격도** | `(현재가 - EMA) / ATR` | 변동성 보정 거리 | 이평선과의 실질적 거리 |
| **롤링 상관계수** | `corr(A, B, window=n)` | 관계의 건강상태 | 금-달러 관계가 살아있는가? |
| **비율 (Ratio)** | `A / B` | 상대 강도 비교 | 금/은 비율, 금/구리 비율 |

### ④ 거래량(Tick Volume) 피처 — 빠트리지 마세요

가격만 보는 AI는 "뼈만 보고 근육을 무시하는 의사"와 같습니다.

- **추가 피처:** `M1 Tick Volume의 20봉 이동평균 대비 현재 비율`
  - 비율 **2.0 이상** → 이 캔들에 "큰 손" 참여 중 (브레이크아웃 유효 확률 증가)
  - 비율 **0.3 이하** → 유동성 고갈 (가짜 돌파 확률 높음, 시그널 무시)
- **IC Markets CFD 주의:** CFD 거래량은 실제 선물 거래량과 다릅니다. 하지만 **Tick Volume(해당 시간 동안 가격이 변한 횟수)**은 실제 거래 활동의 훌륭한 대용치이며, MT5의 `iTickVolume()`으로 바로 접근 가능합니다.

### ⑤ 세션 더미 변수 — 순환 인코딩(Cyclical Encoding) 사용

세션 피처를 `아시아=0, 런던=1, 뉴욕=2`로 넣으면 AI는 "뉴욕(2)은 아시아(0)보다 2배 크다"고 잘못 해석합니다.

- **트리 모델(XGBoost 등)용:** 원핫 인코딩 → `is_asia=0/1`, `is_london=0/1`, `is_newyork=0/1` 3개 더미 변수
- **딥러닝용 (고급):** 시간을 sin/cos 순환 함수로 인코딩

```python
import numpy as np
hour = 16  # KST 기준 현재 시각
hour_sin = np.sin(2 * np.pi * hour / 24)  # -1 ~ +1 순환
hour_cos = np.cos(2 * np.pi * hour / 24)  # -1 ~ +1 순환
# → 23시와 0시가 "거의 같은 시간대"임을 AI가 자연스럽게 이해
```

### ⑥ 모델 앙상블 — 매크로와 마이크로 모델을 분리하라

매크로 피처(H1)는 1시간에 1번 바뀌고, 마이크로 피처(M1)는 1분에 1번 바뀝니다. 갱신 속도가 **60배** 다른 피처를 하나의 모델에 넣으면 AI가 "천천히 움직이는 매크로 피처"를 제대로 학습하지 못합니다.

```
모델 A (매크로 방향 판단기 — H1 기준):
  입력: UST10Y 변화율, EURUSD 변화율, US500, VIX, 상관계수
  출력: "오늘은 Long Bias / Short Bias / Neutral(거래 중단)"

           ↓ Long Bias 일 때만 가동

모델 B (마이크로 타이밍 판단기 — M1 기준):
  입력: RSI 기울기, ATR 이격도, 볼린저 %b, Tick Volume 비율
  출력: "지금 이 1분봉에서 진입 / 대기"

           ↓ 진입 결정 시

피처 중요도 종합 점수 → 랏사이즈 결정 → Triple Barrier 청산
```

> [!TIP]
> **트리 기반 모델(XGBoost, LightGBM)을 1차 테스터로 쓰세요.** 이 모델들은 학습 후 "Feature Importance(%)" 성적표를 뽑아줍니다. 처음에 이것저것 넣고 돌려본 뒤, 기여도 1% 미만인 하위 50% 피처는 과감하게 제거하세요. 딥러닝은 "왜?"를 설명 못하지만, 트리 모델은 명세서를 줍니다.

## 🗄️ 벡터 DB를 위한 피처 임베딩 전략

> **2026-02-23 추가:** 위의 피처들을 벡터 DB에 저장하여 유사 패턴 검색(RAG)에 활용하는 전략.
> 도입 시점: **Phase 3** (Triple Barrier 라벨링 객관화 및 피처 확정 후)
> 상세 로드맵: [[XAUUSD_AI_전략개발_종합_로드맵]] 참조

### 왜 피처를 벡터화하는가?

기존 모델은 **현재 피처 값만** 보고 판단합니다. 하지만 벡터 DB를 활용하면 **"과거에 지금과 비슷한 상황이 있었는데, 그때 결과가 어땠는지"** 를 AI에게 알려줄 수 있습니다.

```
기존 방식:
  현재 피처 10개 → AI 모델 → 매수/매도/대기

벡터 DB + RAG 방식:
  현재 피처 10개 → 벡터로 변환
       ↓
  벡터 DB에서 유사 과거 패턴 Top-K 검색
       ↓
  "과거 유사 상황 20건 중 15건 수익 (승률 75%)"
       ↓
  현재 피처 + 과거 패턴 정보 → AI 모델 → 더 정밀한 판단
```

### 임베딩 대상 피처 구성

확정된 피처를 **하나의 벡터**로 결합하여 저장합니다:

```
벡터 구성 (각 슬라이딩 윈도우당 1개 벡터):

  [매크로 피처 — H1 기반]
    UST10Y 1일 변화율
    UST10Y 5일 변화율
    EURUSD 1일 변화율
    EURUSD 방향 (EMA 기울기)
    US500 일간 변화율
    XAUUSD-EURUSD 상관계수
    세션 (sin/cos 인코딩)

  [기술적 피처 — M1 기반]
    RSI(14) 기울기
    ATR 정규화 이격도
    볼린저 %b
    Tick Volume 비율

  [메타 피처]
    금-달러 롤링 상관계수
    금/은 비율 변화

  [라벨 — 벡터 메타데이터로 저장]
    Triple Barrier 결과 (1=수익, 0=손실)
    수익/손실 금액
    세션 정보
    국면 분류 (GO_LONG/GO_SHORT/NO_GO)
```

### 슬라이딩 윈도우 설정

| 윈도우 크기 | 기간 | 용도 | 적합도 |
|-----------|------|------|-------|
| M1 30봉 | 30분 | 단기 패턴 매칭 | ⭐ 기본 권장 |
| M1 60봉 | 1시간 | 중기 패턴 매칭 | 대안 |

> [!NOTE]
> 최적 윈도우 크기는 Phase 1~2에서 소규모 실험(PoC)을 통해 결정합니다. 30봉과 60봉을 각각 테스트하여 유사도 검색 품질을 비교하세요.

### 벡터 DB 활용 단계

```
Phase 3 — 기초 인프라:
  ✅ 확정된 피처 조합의 벡터화 및 저장
  ✅ 유사 국면 검색 → 국면 필터 정밀도 향상
  DB 선택: ChromaDB (로컬 경량) 또는 LanceDB (대용량 시계열 최적)

Phase 4 — 메타 라벨링 강화:
  ✅ 1차 신호 발생 시 유사 과거 패턴 자동 검색
  ✅ 검색 결과(유사 패턴 승률)를 2차 AI 필터 입력으로 활용
  ✅ Follow/Skip 판단의 추가 근거 제공

Phase 5 — 에이전트 메모리:
  ✅ 실전 거래 결과를 실시간으로 벡터 DB에 축적
  ✅ 계층적 메모리(단기/중기/장기) 구축
  ✅ 온라인 학습의 핵심 인프라로 활용
```

### 피처 스케일링과 임베딩의 관계

> [!WARNING]
> **임베딩 전에 반드시 피처 스케일링(정규화)이 완료되어야 합니다.** 위의 "피처 스케일링 가이드" 섹션에서 권장한 롤링 Z-스코어 또는 랭크 기반 스케일링을 적용한 후 벡터화해야 유사도 검색이 의미 있는 결과를 반환합니다. 스케일링 없이 원본값을 벡터화하면 금리(-0.15%)와 RSI(35) 같은 스케일 차이로 인해 유사도 계산이 왜곡됩니다.

> [!CAUTION]
> **데이터 품질 검증 필수:** 결측치·이상치가 벡터에 포함되면 거리 계산이 왜곡됩니다. ETL 파이프라인에 **정합성 검증 단계**(Missing-value check, Outlier clipping)를 반드시 삽입하세요. 특히 매크로 피처(금리, 지수)는 거래 시간 외에 데이터가 없으므로 forward-fill 매핑 시 결측 여부를 확인해야 합니다.
