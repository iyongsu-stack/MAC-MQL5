## 🔺 피라미딩 (Pyramiding) 기법 정리

피라미딩은 **기존 포지션이 수익 중일 때 추가 진입하여 수익을 극대화**하는 기법입니다. 특히 현재 프로젝트의 **트렌드 추종 + TrailingStop** 구조와 매우 잘 맞습니다.

---

### 1. 피라미딩의 핵심 원리

```
가격 ↑
  │         ┌──── 3차 추가 (최소 랏)
  │     ┌───┤
  │ ┌───┤   └──── TrailingStop ━━━━━━━━━━┐
  │ │   └──── 2차 추가 (소 랏)            │ 
  │ │                                     │ 전체 청산
  ├─┤──────── 1차 진입 (기본 랏)          │
  │ │                                     ▼
──┴─┴───────────────────────────────────────── 시간 →
```

**핵심**: 추세가 진행될수록 포지션을 **점진적으로 증가**시키되, 각 추가 진입의 랏 사이즈는 **감소**시킴

---

### 2. 피라미딩 유형

| 유형 | 랏 배분 예시 | 특징 | 리스크 |
|:---|:---|:---|:---|
| **정피라미드** (Standard) | 1.0 → 0.5 → 0.25 | 가장 보수적, 평균 단가 유리 | ⭐ 낮음 |
| **균등 피라미드** (Equal) | 0.5 → 0.5 → 0.5 | 구현 간단 | 중간 |
| **역피라미드** (Inverted) | 0.25 → 0.5 → 1.0 | 최대 수익 잠재력 | ⚠️ 매우 높음 |

> 💡 **권장**: 정피라미드 (Standard). 추세 초기에 가장 크게 진입하고, 추가 진입 시마다 줄여감

---

### 3. 피라미딩 조건 설계 (BSP 프레임워크 맥락)

현재 시스템 구조에 맞는 피라미딩 조건:

```text
[1차 진입] AI 신호 발생 (BSP>1.5 + TV>1.2× + AI 확률 > 0.6)
    → 기본 랏 진입, TrailingStop 활성화

[2차 추가 조건]
    ✅ 1차 포지션이 ATR × 0.5 이상 수익 중
    ✅ BSP Scale이 여전히 > 1.5 (추세 지속 확인)
    ✅ AI 재확인 신호 (선택적)
    → MoneyManageVx가 가용 증거금(Free Margin)과 최신 ATR을 기반으로 동적 랏 사이즈(예: 1차 대비 50%) 계산 후 진입

[3차 추가 조건]
    ✅ 전체 (그룹) 포지션이 총 ATR × 1.0 이상 누적 수익 중
    ✅ BSP Scale > 2.0 (강한 추세 확인)
    → MoneyManageVx가 가용 증거금 기반으로 동적 랏 사이즈(예: 1차 대비 25%) 계산 후 진입

[공통 관리 및 청산 (Hedging 계좌 반영)]
    → MT5 Hedging 계좌 특성상 다중 진입은 각각 별도의 티켓 번호로 생성됨.
    → TrailingStopVx 모듈은 동일한 '매직 넘버(Magic Number)'를 가진 그룹 포지션의 통합 누적 PnL을 모니터링하여 조건 충족 시 일괄 청산.
```

### 4. 피라미딩의 수학적 효과

**기본 시나리오** (피라미딩 없이):
- 1.0 랏 × 100pt 수익 = **100pt**

**피라미딩 시나리오** (정피라미드):
| 진입 | 랏 | 진입가 대비 수익 | 기여 수익 |
|:---:|:---:|:---:|:---:|
| 1차 | 1.00 | +100pt | 100pt |
| 2차 (+50pt 시점) | 0.50 | +50pt | 25pt |
| 3차 (+80pt 시점) | 0.25 | +20pt | 5pt |
| **합계** | **1.75** | | **130pt** |

> 📈 같은 추세에서 **+30% 추가 수익** 확보 (리스크는 TrailingStop으로 통제)

---

### 5. 리스크 관리 핵심 규칙

| 규칙 | 설명 |
|:---|:---|
| **최대 추가 횟수** | 2~3회로 제한 |
| **총 노출 한도** | 전체 포지션 합이 최대 리스크의 2배 미달 (Free Margin 기반 `MoneyManageVx` 랏 제어) |
| **추가 진입 조건** | 매직넘버가 동일한 그룹 포지션 전체가 **반드시 통합 수익 중**이어야 함 |
| **통합 손절선 상향 (B/E)** | 추가 진입 즉시, 전체 그룹 포지션의 **평균 단가(Average Open Price) + 총 마찰비용(30pt × n)** 이상으로 모든 포지션의 SL을 일괄 이동시켜 하락 반전 시 원금 손실을 완벽 방어 |
| **일괄 청산** | TrailingStop 터치 시 그룹 내 **전체 포지션 동시 청산** |

---

### 6. 현재 시스템과의 시너지

현재 구조가 피라미딩에 **이상적인 이유**:

1. **TrailingStop 전담 청산** → 추세가 끝날 때까지 보유, 피라미딩과 궁합 최적
2. **BSP Scale 실시간 모니터링** → 추세 강도 기반 추가 진입 판단 가능
3. **AI 확률 출력** → 추가 진입 시점에서도 AI 재확인 가능
4. **ATR 기반 동적 간격** → 추가 진입 간격을 변동성에 맞게 자동 조절

---

### ⚠️ 주의사항 (Strategic Advisor)

1. **Walk-Forward에서 피라미딩 포함 검증 필수** — 피라미딩 없는 승률과 피라미딩 포함 총 수익을 별도 비교
2. **시스템 모듈 통합 필수** — `OpenCloseVx`(진입 판단), `MoneyManageVx`(동적 랏+증거금 계산), `TrailingStopVx`(그룹 포지션 판별 및 평균단가 B/E 업데이트) 간 유기적 통신 구현이 핵심
3. **Friction Cost 누적** — 추가 진입마다 30pt × 진입 횟수만큼 마찰 비용이 가중되므로 B/E 이동 시 반드시 고려
4. **먼저 피라미딩 없이 기본 전략을 완성한 후** 피라미딩을 별도 모듈로 추가하는 것이 안전

## 🎯 피라미딩 진입 시점 AI 학습

피라미딩 진입 시점도 **AI가 학습할 수 있습니다.** 핵심은 라벨링 방식을 어떻게 설계하느냐입니다.

### 접근법: 멀티 라벨 학습 (Multi-Label)

현재 시스템은 **단일 라벨**(진입 여부 1개)이지만, 피라미딩을 학습시키려면 **별도의 라벨**이 필요합니다:

```
[기존] Model_Entry:   "지금 진입할까?" → label_long (0 or 1)

[확장] Model_AddOn:   "기존 포지션에 추가할까?" → label_addon (0 or 1)
```

### 구체적 설계: 2-모델 구조

```
                    ┌──────────────────────┐
  M1 봉 데이터 ──→ │  Model_Entry (기존)  │ → 1차 진입 신호
                    │  480 피처            │
                    └──────────────────────┘

                    ┌──────────────────────┐
  M1 봉 데이터 ──→ │  Model_AddOn (신규)  │ → 2차/3차 추가 신호
  + 포지션 상태 ──→ │  480 + α 피처        │
                    └──────────────────────┘
```

### Model_AddOn에 추가되는 피처 (α) — 확정 (2026-03-11)

| 피처 | 설명 |
|:---|:---|
| `unrealized_pnl_atr` | (Open봉 - 1차진입가) / ATR (정규화 미실현 수익) |
| `bars_since_entry` | 1차 진입 후 경과 봉 수 |
| `bsp_scale_delta` | 진입 시점 대비 BSPScale(`LRAVGST_Avg(60)_BSPScale`) 변화 (추세 안정성) |
| `atr_expansion` | 현재ATR / 1차진입ATR — 변동성 확장 비율 |
| `trend_acceleration` | BSPScale 5봉 기울기 (추세 가속 중인가) |
| `addon_count` | 이미 추가한 횟수 (0, 1, 2) |

### 라벨링 방법 (label_addon) — 재설계 (2026-03-11)

**2단계 구조** (기존 고정 3-Barrier → CE2 트레일링 파동 기반으로 전면 재설계):

```
[1단계: 파동 판별]
  label_long=1 봉(1차 Win 진입) 기준 CE2 Forward Scan:
  SL(7×ATR) 터치 먼저       → 패배 파동 → SKIP
  CE2 발동, PnL < 4×ATR     → 패배 파동 → SKIP
  CE2 발동, PnL >= 4×ATR    → 승리 파동 확정

[2단계: 봉별 label_addon 판정 (승리 파동 내 봉만)]
  전제: unrealized_pnl > 0.3×ATR (1차 포지션 수익 중)
  WIN  (label_addon=1): (CE2청산가 - Open봉 - 30pt) >  1×ATR
  LOSS (label_addon=0): (CE2청산가 - Open봉 - 30pt) <= 1×ATR
```

**핵심 파라미터 (확정)**:

| 파라미터 | 값 | 의미 |
|:---|:---:|:---|
| `SL_ATR_MULT` | **7.0** | A안: 롱전략 1차 진입과 동일한 SL 공유 |
| `WAVE_MIN_ATR` | **4.0** | 승리 파동 최소 수익 기준 (CE2 발동 시) |
| `ADDON_MIN_ATR` | **1.0** | 봉별 추가진입 WIN 최소 수익 기준 |
| `UNREAL_MIN_ATR` | **0.3** | 추가진입 전제 조건 (1차 수익 중 확인) |
| `FRICTION_PT` | **0.30** | 마찰비용 30pt |

> **WIN 핵심 수식**: `(CE2청산가 - 해당봉 Open - 30pt) > 1×ATR`

**설계 근거**:
- **SL=7ATR (A안)**: 롱전략 확정 파라미터(SL=7, CE~4ATR)와 완전 정합
- **파동 기준 4ATR**: 롱전략 시뮬레이션(분석 섹션13)에서 CE_TP 최적 최소값
- **봉 기준 1ATR**: 파동 내 고점 근처 "늦은 추가진입"을 AI가 학습하여 회피

## 🎯 최적 피라미딩 전략 도출 (2026-03-11)

### 확정된 기반 (진입/청산)

```
진입: AI A+B+C prob ≥ 임계치 (M30 필터 미사용)
SL:   ATR14(Wilder) × 7.0 (고정)
TP:   CE2 트레일링 청산, 수익 < 4×ATR이면 CE리셋 후 다음 파동 대기
청산: CE2 SL 터치 시 전체 포지션 일괄 청산
```

**실전 시뮬 결과 (격자탐색 SL=7, TP=4.0, thr=0.20)**:
- **739건, 승률 77.9%, 기대 5.59pts, PF 3.62**

**피라미딩 모델(Model_AddOn) OOS 결과**:
- AUC=0.78, 승률 **73~78%**, OOS 3/3 PASS

### 최적화해야 할 6개 변수

| # | 변수 | 탐색 범위 | 의미 |
|:---:|:---|:---|:---|
| 1 | `addon_thr` | 0.20 ~ 0.50 | 피라미딩 AI 임계치 (높을수록 보수적) |
| 2 | `max_addon` | 1, 2, 3 | 최대 추가 횟수 |
| 3 | `lot_ratio` | 0.25 ~ 1.0 | 추가 랏 비율 (1차 대비, 정피라미드) |
| 4 | `min_interval` | 5 ~ 30봉 | 추가 진입 최소 간격 |
| 5 | `min_unrealized` | 0.5 ~ 2.0 ATR | 추가 진입 전제 (1차 최소 미실현수익) |
| 6 | `be_mode` | shared / be_move | B/E SL 이동 방식 |

### 피라미딩 리스크 제어 플로우

```
1차 진입: 1.0 랏, SL = 7×ATR (고정, 1차 전용)
                  ↓
     [AI Model_AddOn prob ≥ addon_thr?]
     [미실현 수익 ≥ min_unrealized × ATR?]
     [마지막 추가 후 ≥ min_interval 봉 경과?]
                  ↓ 모두 Yes
2차 추가: lot_ratio × 1차랏, SL → 1차 진입가 + 총 마찰비용 (B/E 이동)
                  ↓ 동일 조건 재확인
3차 추가: lot_ratio² × 1차랏, SL → 전체 평균단가 + 마찰비용 (B/E)
                  ↓
     CE2 발동 (≥4ATR) → 전체 일괄 청산 → 수익 극대화
     또는 SL 터치 → B/E 이동 되었으므로 손실 ≈ 0 (리스크 제로화)
```

> **핵심**: 추가 진입할 때마다 **전체 SL을 B/E(손익분기)로 올려서**, 되돌림이 와도 **1차 수익으로 추가 포지션 손실을 커버**

### 도출 방법 — 3단계

#### B단계: 기대값 분석 (30분) ← 먼저

> "피라미딩 자체가 수학적으로 이득인가?"

- 피라미딩 모델 승률(73~78%) × 평균 Win/Loss PnL → **EV(기대값)** 산출
- Kelly Criterion으로 **이론적 최적 랏 비율** 계산
- 추가 1회/2회/3회별 누적 리스크 비교

#### C단계: 승리 파동 프로파일링 (1시간)

> "파동 안에서 가장 좋은 추가 시점은 어디인가?"

- `labels_pyramiding_full.parquet`에서 승리 파동(CE2≥4ATR) 분석:
  - 파동 지속 봉수 분포 → `min_interval` 가이드
  - unrealized_pnl 구간별 추가진입 성공률 → `min_unrealized` 가이드
  - 파동 내 최대 되돌림 분포 → B/E SL 마진 가이드
  - 1파동 내 label_addon=1 봉 수 분포 → `max_addon` 가이드

#### A단계: 통합 CE+피라미딩 시뮬레이터 격자탐색 (2~3시간) ← 최종

> "B+C 결과로 좁힌 범위 안에서 최종 파라미터 확정"

- 기존 CE 시뮬레이션 + 피라미딩 로직 통합
- 6개 변수 격자탐색 (B+C에서 범위 축소)
- 측정: 피라미딩 vs 미적용 **수익 증가율**, **추가 포지션만의 PnL**, MDD, Calmar

## 💰 랏사이즈 공식 — 피라미딩 포함 총 리스크 1% 제한 (2026-03-12)

### 핵심 공식

```
worst_case_ATR_lots = SL_mult × 1.0
  + Σ (SL_mult + spacing × n) × lot_ratio^n   (n = 1..max_addon)

base_lot = (Balance × risk%) / (worst_case_ATR_lots × ATR14 × contract_size)
```

**기본값 대입 (SL=7, addon 3회, ratio=0.50, spacing=1.5ATR)**:

| 포지션 | SL거리(ATR) | Lot 비율 | ATR-lots |
|:---|:---:|:---:|:---:|
| Base | 7.0 | 1.000 | 7.000 |
| Addon 1 | 8.5 | 0.500 | 4.250 |
| Addon 2 | 10.0 | 0.250 | 2.500 |
| Addon 3 | 11.5 | 0.125 | 1.438 |
| **합계** | | | **15.188** |

> **해석**: base만 1%로 계산하면 worst-case 총 손실이 ~2.2%. 이 공식은 **전체 피라미딩 그룹이 SL 히트 시에도 정확히 1% 손실**이 되도록 base_lot을 역산.
