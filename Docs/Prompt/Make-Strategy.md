## 2. 핵심 발견사항 (프로젝트에서 검증된 사실)

### 🔑 가장 중요한 발견: 알파는 청산에서 온다

| 청산 방식 | PF (2025) | 결론 |
|:---|:---|:---|
| 대칭 1:1 (TP=SL) | 0.99 | ❌ 손익분기 |
| 비대칭 2:1 (TP=2×SL) | 1.01 | ❌ 승률 하락으로 상쇄 |
| **트레일링스탑** | **1.23** | ✅ **유일한 알파** |

> **결론**: 어떤 진입 조건을 써도 PF ~1.0. 트레일링스탑만 알파를 만든다.

### 지표 분석 결과

| 지표 | 역할 | 최적 사용법 |
|:---|:---|:---|
| LRA(180) | 장기 추세 필터 | `> 0` 일 때 Long Only |
| ADX(14) | 추세 강도 | `> 25` (횡보 필터) |
| LRA(60) | 단기 모멘텀 | `> 0` (타이밍) |
| BOP_Smooth | 매수/매도 압력 | `> 0` (매수세 확인) |
| RSI(14) | 과매수 방지 | `< 70` |

### 가속도 분석 (2차 미분)

```python
Accel = 현재값 - N봉전 값  (N=10)
```

- 모든 지표에 적용 가능 (LRA, ADX, BOP, RSI, CHV, CSI, TDI, QQE)
- **진입 필터**: `LRA_Accel > 0 AND ADX_Accel > 0 AND BOP_Accel > 0`
- 의미: 기울기 강화 + 추세강도 증가 + 매수압력 증가

## 3. 종합 분석 8가지 (반드시 수행)

| # | 분석 | 왜 필요한가 | 전략 활용법 |
|:---|:---|:---|:---|
| 1 | **다이버전스** | 추세 반전 조기 감지 | 진입 금지 조건 |
| 2 | **조합 클러스터링** | 어떤 조합이 수익성 있는지 | 최적 진입 필터 선택 |
| 3 | **변동성 레짐** | 시장 상태별 전략 효과 차이 | SL/TP 동적 조절 |
| 4 | **지표 분포** | ADX>25가 최적인지 검증 | 임계값 데이터 기반 설정 |
| 5 | **세션 분석** | 시간대별 수익성 차이 | 시간 필터 최적화 |
| 6 | **R-Multiple** | 청산 품질 평가 | 트레일링 거리 조절 |
| 7 | **리드/래그** | 어떤 지표가 선행하는지 | 선행 지표로 조기 진입 |
| 8 | **ML 피처 중요도** | 불필요한 지표 제거 | 모델 간소화 |

## 4. 검증된 챔피언 전략 사양

### 진입 조건 (LONG Only, 모든 AND)

```python
signals = (
    (LRA_Long > 0) &        # 장기 상승 추세
    (ADX > 25) &             # 추세 강도 충분
    (LRA_Short > 0) &        # 단기 모멘텀 회복
    (BOP_Smooth > 0) &       # 매수세 우위
    (RSI < 70) &             # 과매수 아님
    (LRA_Accel_S > 0) &      # 기울기 가속
    (ADX_Accel > 0) &        # 추세강도 강화
    (BOP_Accel > 0)          # 매수압력 증가
)
```

### 청산 조건 (Trailing Stop)

```python
SL_PTS         = 2.0   # 초기 스탑로스 (포인트)
TRAIL_TRIGGER  = 1.0   # 수익 1pt 도달 시 트레일링 시작
TRAIL_DISTANCE = 1.0   # 최고점에서 1pt 뒤 추적
MAX_BARS       = 120   # 최대 보유 2시간
```

### 고정 파라미터

```python
P_LRA_SHORT    = 60    # 단기 선형회귀
P_LRA_LONG     = 180   # 장기 선형회귀
P_ADX          = 14    # ADX 기간
P_RSI          = 14    # RSI 기간
P_BOP_AVG      = 50    # BOP SMA 1차
P_BOP_SMOOTH   = 20    # BOP SMA 2차
P_ACCEL_LOOKBACK = 10  # 가속도 룩백
```

## 6. 과적합 방지 원칙

1. **In-Sample / Out-of-Sample 분리**: 2026 데이터로 최적화 → 2025 데이터로 검증
2. **장기 검증 필수**: 최소 10년 (2010~2020) 데이터에서 양의 PF 확인
3. **파라미터 수 최소화**: 지표 기간은 고정, 조절은 SL/TP/트레일링만
4. **교차검증**: Optuna 5-Fold CV로 안정성 확인
5. **가중합 경계**: Weighted Sum은 과적합 위험 → AND 논리가 안전

## 9. MQL5 코딩 시 핵심 규칙

1. **BSP Framework 모듈 구조** 준수 (`Include/BSPV9/`)

3. `GetLastError()` 필수 → 모든 거래 명령 후 에러 처리
4. `ExternVariables.mqh` 최우선 Include
5. 한국어 주석 필수, 변수명은 CamelCase
